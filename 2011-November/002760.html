On Sun, Nov 13, 2011 at 11:36 AM, Joshua Beck <mikeosdeveloper at gmail.com> wrote:
>
> Can you add os_command_line as a system call after the
> existing ones in beta 7, so my program will work?

Not in its current state, no. Exposing this API to external programs
is potentially a huge source of problems. os_command_line loads
programs at 32768, so what happens if RANDOM.BIN calls
os_command_line, which then loads a new file over where RANDOM.BIN is?
Where do we "ret" back to in the end? It can get very messy.

So this would be a bad move, in terms of design. It's far more
sensible if, one day, we have a completely independent
os_command_parse which any code (including the kernel's own CLI) can
use. Will think about it for 4.4.

Mike
