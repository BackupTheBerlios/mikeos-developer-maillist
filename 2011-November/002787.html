Hey folks,

To explain my disappearance from the list as of late, I've been working 
on a couple things, including some nifty little programs for MikeOS. But 
the largest usage of my OS development related time has been getting 
Blacklight OS on its feet.

Blacklight OS is a simple, singletasking real mode x86 OS that is going 
to end up somewhere between a DOS and MikeOS. Today I managed to port 
the MikeOS FAT12 driver to the OS, set up a far call based syscall API, 
and port one of my MikeOS programs (Extended CPUID) to it with minimal 
effort.

Currently, the kernel loads at 1000:0000, connects to the Advanced 
Power Management (APM) interface if it can find one, searches for 
CPUID.BIN, loads it to 2000:0000 in two stages (first the file is read 
and loaded to 1000:8000 using fat12_file_load, then the file is 
relocated to 2000:0000), sets up segments for it, and executes it. When 
the program returns with a retf instruction, the kernel restores all the 
segment registers to 1000h, and prompts the user to press any key to 
shut down the computer via APM.

Here's a screenshot of Extended CPUID running under both Blacklight OS 
and MikeOS 4.3b7 (56k warning, 300 KB PNG): http://i.imgur.com/lfxGD.png

I realize this isn't much of an accomplishment in the grand scheme of 
things, but I find it kinda cool.
