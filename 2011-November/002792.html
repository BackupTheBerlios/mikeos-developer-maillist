> Currently, the kernel loads at 1000:0000, connects to the Advanced 
> Power Management (APM) interface if it can find one, searches for 
> CPUID.BIN, loads it to 2000:0000 in two stages (first the file is read 
> and loaded to 1000:8000 using fat12_file_load, then the file is 
> relocated to 2000:0000), sets up segments for it, and executes it. When 
> the program returns with a retf instruction, the kernel restores all the 
> segment registers to 1000h, and prompts the user to press any key to 
> shut down the computer via APM.

I'm not completely sure what you mean by 'APM' (Advanced Power management)... but I have built using the MikeOS bootloader a miniture DOS like thing in 512 bytes... I'm using it as the base to my project.
 
What it can do:-
 - Automatically trys to run 'AUTORUN.BIN'
 - On failure to do so it offers a dos like prompt, where programs can be typed in (automatically adds the extension of '.BIN'.)
 - Offers programs the ability to print to screen (0x0000:0x7C00 + 1AEh) and the ability to do l2hts conversions (Can't remeber the call location).
 - Loads programs/kernels to 2000h:0000h
 - Programs/kernels can jump back to the bootloader at 0x0000:0x7C00.
 
It's not the most brilliant thing in the world, but with little adjustement it might be able to offer a 'backup' type service, or give you the ability to run other programs once your management has been run.
