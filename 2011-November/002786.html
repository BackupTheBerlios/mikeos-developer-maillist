Hi Dan,

There seems to be some ISR functions under int 15h (AX=C200 through 
C209) that handle setup and initialization of the mouse in a way the 
BIOS feels comfortable with. I'd advise against using C208 and C209 
though since they're not supported by all BIOSes (IBM classifies them as 
optional). Maybe you could find some way to use these to set up a 
"driver" of sorts instead of continually polling the keyboard (which I 
assume takes a fair bit of time in comparison to waiting on an interrupt 
of sorts.)

Information on how stuff is passed to the driver via the stack: 
http://www.ctyme.com/intr/rb-1603.htm

I'm also seeing if I can come up with a solution for your problem. I'll 
let you know if I find anything. In the meantime, go nuts!
