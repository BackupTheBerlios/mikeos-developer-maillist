Hi Raghav,

FASM and NASM are similar syntax.
It's basically the directives and macro system which are different.
There are no macros in PM32 MikeOS. To convert the source to NASM
simply change DEFINE and INCLUDE to %DEFINE and %INCLUDE.

> I will start with the PS/2 keyboard driver.

Originally my goal with aeBIOS was to handle all the standard AT BIOS
functions. The current version only handles the most used subset. The
reason for this is that I intend to provide PM32 (and later on 64BIT)
code for the aeBIOS functions. This way those that are interested in
driver development can contribute reusable code which can be used by
hobby OS developers through the standard BIOS interface. This code can
be developed with a working PM32 system using a working and developed
OS. There is no need for the original MikeOS developers to change
directions as their excellent work can be easily translated to PM32.
