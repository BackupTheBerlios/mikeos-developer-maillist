Hi Scott,

> I saw somewhere translated to a 32-bit protected mode version of MikeOS.

That's using a special BIOS implementation from Mike Gonta. It's very
cool but it's not a standalone protected mode OS.

> think everyone would want to be in protected mode?  Unless it causes grief
> using the BIOS interrupts?

Correct. We cannot use BIOS routines from protected mode. Currently we
use the BIOS for screen output, keyboard input, disk drive
reading/writing etc. Without the BIOS, we'd have to write very picky,
complex drivers for each of those devices, and MikeOS would become
MUCH more complicated.

Right now, MikeOS shows you how to do the interesting things in an OS
-- create system calls, execute programs etc. I never want thousands
of lines of complex hardware driver code.
