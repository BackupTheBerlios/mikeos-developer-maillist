> Actually it's running directly on a standard BIOS interface which runs
> on the bare metal. aeBIOS has the same interface (onlt PM32).

Oh absolutely, and it's awesome that something like MikeOS can use
aeBIOS without a large amount of changes to the code. I was just
thinking of how I started with MikeOS: I wanted to write something
that used a 'raw' PC -- no need to load anything else, no libraries or
add-on facilities. Just me and a freshly powered-on PC.

I'm wondering if we could use aeBIOS for a kind-of "super MikeOS" like
we've talked about in the past on this list. I still think there's
scope for a broader project that's not tightly focused on simplicity
and teaching like MikeOS, but has room for more features and
experimentation.

I'd love to do something like that, and aeBIOS would be a huge help in
giving us loads of RAM to wallow in. Then roll in new features: a
mouse driver, higher-res PCX viewer, and anything else that could make
use of protected mode. I don't have a lot of time for coding at the
moment, but if anyone wants to make a serious go of this, that'd be
excellent!

M
