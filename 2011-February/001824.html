A serious app like this...

I developed an independant Python Enhancement project for the classic
AMIGA to use a Python Shell as a GUI:-

http://aminet.net/package/dev/src/Python_Enhancement

It uses independant executables to generate the GUI and use it...

This runs on a STANDARD A1200.

In MikeOS`s case nice front ends could be used for serious apps like that
long term Data-Logger, say in 320x200 MCGA emulation...

If you wanna see the python code it is here:-
http://homepages.tesco.net/wisecracker/public/DL.py

So complex cool looking front ends could be done using separate executables
each run separately, from say MikeBASIC used as a script interpreter... ;oO

See the python code for the basic ideas...

I`m not sure if I am lucid enough but it releases MikeOS from a dedicated GUI
to a list of pure executables to paint the necessary bits onto screen and the
INTs to plot the graphs...

All this could be done easily using MikeBASIC.

A convoluted idea? Yes, but could be made to work on MikeOS as all you will be
doing in this case is writing directly to display memory for the main "GUI" ONCE
but ANY, (IF ANY), moving bits repeatedly called from disk as required...

This front end for example is 320x200 32 colours only in its original form.
Mouse control is NOT needed for this either:-

http://homepages.tesco.net/wisecracker/public/SCOPE.GIF

Even with MikeOS`s restricted environment this could be made functional.

REMEMBER the mouse is NOT a requirement for this idea to work.
