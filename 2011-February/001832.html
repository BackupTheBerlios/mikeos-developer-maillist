(I've also split up Downloads into sections for add-on software and
Bazza's hardware projects.)

Ondrej, I spotted a couple of bugs:

1) Run BFM, run EDIT.BIN inside that, open EXAMPLE.BAS, hit F8 to run
BASIC and then exit the BASIC program. Hit Esc to leave EDIT.BIN and
strange stuff happens. Presumably this is to do with the complicated
memory arrangements.

2) In the BFM editor: open EXAMPLE.BAS and go to the start of the
"cls" line. Hit backspace and it removes the blank line before (good),
but it moves the cursor to the end of the "cls" line (when it should
remain at the start).

I like your editor though, and it seems to handle long lines better
than my EDIT.BIN! (Mine doesn't handle backspace to remove lines
because that'd involve the cursor jumping back to the end of the
previous line, which could be longer than the screen, which I don't
cater for yet).

It'd be great to have a more powerful editor for MikeOS.
