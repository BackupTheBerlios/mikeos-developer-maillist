Actually, aeBIOS is a BIOS extender.
All PM32 BIOS calls made in the PM32 translated MikeOS are executed
by the native motherboard 16 bit real mode BIOS.
Unlike DOS extenders there is no hosted environment, aeBIOS
does not run under an existing OS. There are no special calls or
setup to do. aeBIOS takes over in a way similar to the video and hard disk
option roms, aeBIOS is software based and not permanent.

RM16 MikeOS story:
The native RM16 BIOS loads someone elses non MikeOS boot sector code
which then loads RM16 MikeOS (generally the boot sector is not considered
to be the OS). Then RM16 MikeOS interacts with the BIOS.
RM16 BIOS <-> RM16 MikeOS

RM32 MikeOS story:
The native RM16 BIOS loads aeBIOS which then loads PM32 MikeOS. Then
PM32 MikeOS interacts with the BIOS.
RM16 BIOS <-> aeBIOS <-> PM32 MIkeOS
