Do you mean like the propietary real mode BIOS?
aeBIOS has the same interface, you stuff the registers and make the int 
call.
There is no special setup or special **wrapped** interrupt calls.
The differences are slight and directly related to PM32.
#1  word size is 32 bit, so any word size registers (ax, bx, etc.) are now
    32 bit (eax, ebx, etc.)
#2  segment registers are not used to pass parameters. All addresses are 
flat 32
    bit (aeBIOS takes care of translating and buffering). The default aeBIOS
    mode is single segment flat address (the same simple setup used by 
Windows and Linux).
    In this PM32 mode the segment selectors are not modified.
