Another bizarre idea for MikeOS and in keeping with its simplicity.

How about re-vectoring INT 18H towards code in segment 0 for MOS usage
only.

It normally points to ROM BASIC from the XT days, with an extension to
INT 19H if ROM BASIC is not found.

I`ve been doing some experiments re-vectoring and will post some test code
as INT18.BIN to run from MikeOS as a command ASAP.

We would have the capability of over 200 register AH offsets that we could
design for ourselves down inside segment 0.

It could be loaded at 0x0000:1000.

I will do a DMA "Hello World..." vector using AH as 0 and AL as 0...
