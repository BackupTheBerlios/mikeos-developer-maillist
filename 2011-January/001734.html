> What does the ORG instruction actually means?

In NASM org tells the assembler what offset to apply to code
labels. In other words it is a hint to the assembler as to
where you intend to load the flat binary code.

> I've made a bootloader to load the kernel into 1000h:0000h 
> 1000h is the segment and 0000h is the offset.

The segment register value is multiplied by 16 and the offset
is added to obtain the absolute physical address in real mode.

> After setting all the correct parameters for loading a file
> on a disk, I used jmp 0000h:1000h instead of 1000h:0000h
> and It won't load the file. (it will load if it is the reverse).

The file is still loaded correctly to absolute address 10000h,
however jmp 0:1000h is absolute address 1000h and they are not
the same.

> I've tried settings ORG to different values and the kernel
> is still being loaded. wasn't it supposed to define the segment?

In 16 bit real mode the "org" defines the offset within the 64K
segment, so it should be the same as the loaded offset,
in this case 0.
