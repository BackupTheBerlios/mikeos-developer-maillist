I wasn't thinking about that.
In real mode there are "short" 2 byte jumps and "near" 3 byte jumps
depending on the distance from the jump instruction to the label.
If the jump table is less than 128 bytes the first jump will be
2 bytes and that will throw off the equ's. Simply force the first
jump to be 3 bytes (the others should be ok)

  jmp NEAR System.Initialize      ;0000h

>> What does align means and why MikeOS don't use align and org 0?

"align" is used to locate the instructions on a boundary (by padding
with nop instructions) for efficient code execution.
MikeOS doesn't use it.
"org 0" is the default and is not required.
