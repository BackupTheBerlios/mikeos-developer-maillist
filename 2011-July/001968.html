I ran the MEMVIEW.BAS in the BASIC interpreter which is part of KERNEL.ASM
with maximum optimization and it works fine on WINXP with qemu.

One interesting thing I noticed is that the 9 byte difference between the
FASM assembled code and the NASM assembled with maximum optimization is
that NASM assembles the unnecessary DS override (unnecessatry since DS is
the default), while FASM does not.

Of course it also works as advertised on a real PC floppy disk and a
real PC USB flash drive (two different PC's).

On a side note, MikeOS will only boot and run with a USB flash drive on
older computers which support USB floppy disk emulation, as well as on
some older computers which support USB floppy disk emulation with a small
capacity flash drive under 1Gb.
Newer computers boot all USB flash drives with hard drive emulation.
MikeOS is hard coded for floppy disk parameters only.
A USB flash drive with a floppy disk image can boot and run with hard
drive emulation just fine without the hard coding. The PM32 port of
MikeOS is a good example.
