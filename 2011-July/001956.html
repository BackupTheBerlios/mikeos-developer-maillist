OK OK OK, I've discovered the problem. NASM 2.09.04 doesn't build the
MikeOS kernel correctly. I downgraded to 2.05, and it builds a kernel
that's much closer to the one in Josh's floppy image (just two bytes
smaller), but crucially, it runs Josh's program.

So the short term solution: beware NASM versions newer than 2.05!

However, I can't believe a mature project like NASM could suddenly get
really bugged, so perhaps there's something in the MikeOS code that
needs tweaking. A nice prize to whoever finds the fault!
