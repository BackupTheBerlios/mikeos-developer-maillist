Don't even consider it!!!

Read the docs first...

MikeOS uses two segments, 0x0000:0000 and 0x2000:0000, (PLUS, if you include
the half segment 0xB800:0000), that is all you have to play with.

Segment 0x0000 is used for the SP, half segment 0xB800 is CGA DMA video ram
and 0x2000 is split between lower 32KB, mikeOS "ROM", and user space in the top
32KB...

I've already uploaded ideas how to exploit these in assembly code but don't attempt
under MikeBASIC; system crashes will undoubtedly occur as it is difficult to cross
pollinate between segments, especially CS and SS, DS and ES are a possibility but
would be tricky under BASIC without serious HW knowledge...

I've also suggested how to re-vector the - now - none existent ROM BASIC INT 18H
for your own usage...

Search back through this list for the stuff...
