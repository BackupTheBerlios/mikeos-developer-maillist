> If I'm understanding correctly, "cli" disables the "int" instruction?
> If that's correct, why is that done?

It's just done while we alter the location of the stack pointer. If an
interrupt occurred and execution jumped elsewhere, the return address
would be pushed onto the stack. If we then go about changing the stack
location during the interrupt service routine, we'll lose that return
address.

This scenario is extremely unlikely to happen here, but it's just good
practice to disable interrupts temporarily while fiddling with the
stack.
