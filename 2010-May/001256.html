Hi Paulo,

I think this is due to Windows interfering with the drive after you've
'dd'ed the image to it. I noticed this a while ago. I made a USB drive
boot MikeOS by 'dd'ing the image in Linux, and booted it up -- it
worked perfectly.

However, I wanted to retrieve a .BAS file from the USB key, so I
plugged it into a Windows box and sure enough, it mounted. However,
when I went to boot the key again, it failed (I think with the 'kernel
not found' error).

So it works under Linux because you umount the drive first and then
'dd' it. Windows, however, can sometimes interfere, change something
on the drive and stop it from booting. I don't know if there's a
workaround other than to suggest booting Linux (at least in live
mode).

Dan: this is nothing to do with USB drivers in MikeOS. The BIOS reads
the USB key as if it were a floppy image.
