>> You do not need an MBR to boot a hard drive and a FAT12
>> formated hard drive boots fine.
> 
> I don't understand. If you have a FAT12 formatted hard drive in a
> machine with no MBR, what does the BIOS load when the machine starts?

The same thing it loaded before the invention of the MBR.
The BIOS loads and runs the first sector on the physical
drive - logical block address 0. On a partitioned hard drive this
is the MBR, the BIOS doesn't care.

Windows only recognizes the first partition on flash drive. So an MBR on
a flash drive is redundant on windows, why use two sectors of code to do
what one can?
