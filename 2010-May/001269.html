>> When using dd to write to a flash drive first copy dd.exe and rename it 
>> dd-removable.exe.
>
> I did that and dd complained about it.

dd-removable is only used to write to a flash drive.
It will not read from a flash drive or write to a floppy disk.

>> Unfortunately MikeOS is hard coded (in many places) to assume floppy disk 
>> geometry.
>> This prevents it from being run from a flash drive which boots on a BIOS 
>> which uses
>> hard drive emulation. See the code in pdBIOS32, specifically boot.asm 
>> which allows
>> it to boot and run under either setup.
>
> Interesting point.
> With all machines I have tried the flash drive on, the BIOS must be set to 
> boot off USB-HDD
> and not USB-FDD otherwise not even the boatloader runs.

USB-HDD refers to a physical USB external hard drive, not a flash drive.
USB-FDD refers to a physical USB external floppy disk drive, not a flash 
drive.
This BIOS can only boot from USB with floppy disk emulation.
