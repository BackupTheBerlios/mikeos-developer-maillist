Perhaps you are right as then it will "release" INT 15h altogether.
I've just discovered that INT 15h is now being used for the SMBUS (system management bus)
so who knows what they've done with all the sub-functions.

Have a look here:

http://smbus.org/specs/smbb10.pdf

and here:

http://en.wikipedia.org/wiki/System_Management_Bus

So bottom line, using INT 15h for WAIT is a no-no, although this SMBUS does open up new possibilities
for monitoring things like battery level, temperature, etc. using MikeOS.
