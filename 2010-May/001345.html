Hi Bazza,

>And AFAIK you can`t set dx to start at 0 either on each call of the 
interrupt.

One does not have to reset anything as it's only necessary to compare tick count and not worry
about absolute values.

The way to check if DX has "overflowed" is first to also store BX (the high order count) when getting the first tick count.
Then when checking the tick count for the second time, we also check to see if BX has remained the same.
If it the same, then it's fine, however if it has changed then ignore the first tick count and get it again subtracting the time (if any) that has already elapsed.

Hope it's making sense.

Paulo.
