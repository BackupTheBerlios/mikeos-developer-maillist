> I did suggest keeping it undocumented FTTB though.

Well, if we were to add os_key_flags, it'd be a new part of the API
and therefore have to be in MikeOS 4.2 (whenever that is). And there
will NEVER be anything undocumented in MikeOS (at least, not
deliberately!)

Undocumented calls and features cause a huge amount of hassle and
cruft in other OSes. As MikeOS is a learning tool, I think we should
lead by example. Everything that's there should be there for a reason,
and everyone should be able to learn what it does.
