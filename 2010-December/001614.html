...yeah...I figured this out a few minutes after asking this.

start:
     mov ax, filename
     mov bx, data
     mov cx, 4192
     call os_write_file

     filename db "file.txt",0
     data db "Hello, World!",0

The string 'data' just gets terminated by more than one zero.  I'm
assuming that MikeOS uses ASCII, so "Hello,World!",0 would be 14
bytes?

I'm spending a lot of time learning assembly, and I just
learned/figured out that 'times' is multiply (pretty obvious, but
still different than a compiled language).

> os_write_file is very simple: you point AX to a zero-terminated
> filename string; BX to a location in the MikeOS 64K of RAM; CX for the
> amount of bytes you want to write (starting from the BX location) and
> then call the routine.
>
>> So where, if in an application, would you write the text to the file, or would

>> work?
>
> Yes, but you'd have to change CX for the appropriate number of bytes.
