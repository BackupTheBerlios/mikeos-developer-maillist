You should really get to grips with assembl(er)(y) first by experimenting
ON top an OS not IN an OS.

Take MS-DOS for example that has Assemblers that run ON top of it.

E.G. the ad86 combination.

Then try writing code to work under MS-DOS using MikeOS restrictions
so that the tools are always available help in your bug tracing.

Make the programs simple so as to understand what is going on.

Even IF the program IS working use MS-DOS`s DEBUG.EXE and step
through the code line by line to see what all of the registers are doing.

This might be a guideline:-

DATA REGISTERS.
AX == Accumulator.
BX == Base.
CX == Counter.
DX == Data.

POINTER + INDEX REGISTERS.
SP == Stack Pointer.
BP == Base Pointer.
SI == Source Index.
DI == Destination Index.

SEGMENT REGISTERS
CS == Code Segment.
DS == Data Segment.
SS == Stack Segment.
ES == Extended Segment.
(Note, FS and GS do not exist in PURE REAL MODE only from 80286 and above.)

OTHER REGISTERS.
IP == Instruction Pointer, (PC == Program Counter in 680x0.)
Status Flags == Note, a very important register to observe on conditional
branching when debugging.

IGNORE all MS-DOS interrupts as MikeOS has no concept of them.

LBNL, don`t run before you can walk.

Hope this helps.
