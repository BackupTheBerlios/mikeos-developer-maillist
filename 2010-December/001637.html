> With that knowledge, I would also like to add a directory program which
> shows all the files on the disk.
>
> I know that it is something to do with Int 13h, and ES and BX, but that is
> it??

Oooh, no way! It's a LOT more complicated than that. int 13h simply
reads and writes sectors (chunks of data) to the disk. It has
absolutely no concept of a filesystem. FAT12 isn't the most
complicated FS in the world, but there's some weirdness (because it
uses full bytes and half of other bytes). You'll need to understand
FAT, the root directory, clusters and more.

Here's a great reference for int 13h:

http://www.ctyme.com/intr/int-13.htm

Here's what I used to implement FAT12 support on MikeOS:

http://www.dfists.ua.es/~gil/FAT12Description.pdf

It's detailed, but you'll need to read all of it to understand how it works.
