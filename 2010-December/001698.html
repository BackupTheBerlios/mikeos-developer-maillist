On 26 Dec 2010, at 06:32, Matthew Caswell <caswellrienzo at gmail.com> wrote:
> 
> The bootloader would most likely have to be modified to boot from a
> USB and/or hard drive, right?

Matt, think before posting. Do a bit of research at least.

1) There is only one bootloader (source/bootload/bootload.asm)
2) MikeOS ALREADY boots from USB

So why would it have to be modified to do something it already does? Your question makes no sense.

> And at kernel level, do the filesystems work the same on a USB FlashFS?

What does this mean? "on a USB FlashFS" -- do you mean a filesystem created specifically for USB flash drives? How would a filesystem work on another filesystem? MikeOS supports FAT12. Almost all USB keys use FAT32. MikeOS runs from a USB key via BIOS floppy emulation, but for native USB key FAT32 support it would need another FS driver and a USB stack.

Matt, you've asked various questions on the list that are answered in the docs, an easy Google away or just make no sense. No more. I know you're new to all this, but you've really jumped in at the deep end without building up some basic knowledge first. You're welcome here to ask questions about MikeOS that aren't answered in the docs - but do proper research and understand the terminology you use first. OS dev is an advanced subject.
