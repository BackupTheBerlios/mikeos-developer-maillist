And another thing. Using os_print_string to display the contents of a
text file is a bad idea. As the API clearly states, os_print_string
takes a zero-terminated string. Your text file, as you described it,
will end in 'T', or possiblly a newline character. NOT a zero.

os_print_string will keep working until it finds the first zero in
RAM, which could be directly after the file contents, or not for
hundreds of bytes after (from previous data). Think about how memory
works.
