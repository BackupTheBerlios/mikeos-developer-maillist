Hi all,

MikeOS 3.3 is here. The biggest new feature in this release is
EDIT.BIN, a full-screen text editor. It's not quite Emacs -- think
more along the lines of Notepad :-) But you can open, edit and save
text files on MikeOS, which could open the door to more programs.

Screenshot: http://mikeos.berlios.de/images/shot-4.png

Note that it currently uses with Unix-style text files; run 'dos2unix'
on a Linux box to convert DOS-style files.

Also in this release: the command line interface can now display text
files using 'cat filename', while the music keyboard (KEYBOARD.BIN)
has a new interface: an ASCII representation of a real keyboard.

There are also some other changes, as detailed in the ChangeLog below.

As always, it's available from http://mikeos.berlios.de

Tomasz: I'll update the site with news of your 0.3 release.
Takayoshi: If you fancy updating the C library, it just needs a hook
for the os_string_parse system call.

Cheers!
Mike


# New program: EDIT.BIN -- a full-screen text editor (currently
limited to Unix-formatted text files)

# README.TXT is no longer copied to the disk images; instead,
programs/sample.txt is added for use in EDIT.BIN

# CLI now includes CAT command to display a file: 'CAT FOO.TXT'

# Music keyboard (KEYBOARD.BIN) given new interface to show a visual
representation of a music keyboard

# os_write_file now removes an older version of the specified file, if
it exists, before performing the write operation

# CLI code (source/features/cli.asm) cleaned up -- split into sections
with fewer local labels

# os_string_parse call is now accessible to external programs

# Disk calls now convert a filename to uppercase automatically, so
there's no need for the calling code to do it

# API version bumped to 11
