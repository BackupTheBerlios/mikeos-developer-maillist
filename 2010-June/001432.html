Hi guys...

Before we start remeber I am NO coder I am an engineer whom can code - BADLY.

I`ve been quiet because I wanted to write positon independant code WITHOUT
using ORG, (Origin).

It all revolved around the Instruction Pointer, IP...

Searching the net provided no clues so eventually I thought of this idea for
MikeOS usage...

I needed it to store data within the constraints of MikeOS BUT have a binary
RUN ANYWHERE when called.

The code below is an example idea only, my finished code for the KB
experiment will follow...

 pushf ;usual stuff here
 pusha ;and here...
 <push........> ;etc.

 call ipfetch ;this is where I find the ip.

 jmp carryon ;this carries on with the code per se.

 nop ;* these NOPs are my storage area.
 nop ;for a relative short jump there is the possibility of up
 nop ;to 120+ bytes of NOP.
 nop ;

ipfetch:
 pop bp ;I now have the offset to the NOPs relative to JMP above.
 push bp ;so long as you remember to push bp and pop bp it can
 ; be reused but still retain the offset to the NOPs relative to the JMP above
 ret

carryon:
 ALL other code from here.
 mov [bp+<JMP offset>+<another number offset as required>],al ;say, this
 ;pokes a value needed to one or more of the NOPs as required.
 ;exitting the code
 <pop........> ;usual stuff here
 popa
 popf
 ret

Hope this lot makes sense.
