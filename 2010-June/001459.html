Hi Matej,

> Seriously, these special cases make no sense to me. As if anyone is actually
> going to feed it zeros.

Perhaps not explicitly, but if your routine is being used in a loop
somewhere with values moving around all the time, you can never be
sure what's going to happen. The worst thing is to not cater for such
behaviour or have different results each time.

If you're not sure what to do in such cases, you could always call
os_fatal_error (with string location in AX). It's quite severe, but it
removes any ambiguity!
