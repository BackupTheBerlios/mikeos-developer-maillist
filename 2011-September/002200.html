You got it Dan, it's VGA.
All computers since about 1999 support what is known as DDE/EDID.
Basically it's I2C on dedicated pins on the vga (see attached pic).

It is normally used for the O.S. (Linux and Windows) to communicate
with the screen to get features, model number, capabilities, etc etc
Since MikeOS does not make use of it and it's I2C, we have it at our
disposal.

To see more about it:

In Linux run get-edid (normally in /usr/sbin/get-edid)
In Windows I guess you would use Control panel then system then Hardware
In MikeOS you would use BIOS (which is what get-edid does):

INT 10 - VESA VBE/DC (Display Data Channel) - INSTALLATION CHECK /
CAPABILITIES

 AX = 4F15h
 BL = 00h
Return: AL = 4Fh if function supported
AH = status
00h successful
 01h failed

INT 10 - VESA VBE/DC (Display Data Channel) - READ EDID

 AX = 4F15h
 BL = 01h
 CX = 0000h
 DX = 0000h
 ES:DI -> 128-byte buffer for EDID record
Return: AL = 4Fh if function supported
 AH = status
 00h successful
 ES:DI buffer filled
 01h failed (e.g. non-DDC monitor)
