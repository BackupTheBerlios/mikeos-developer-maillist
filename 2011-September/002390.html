As I stated:

Some simple guidelines must be followed.

1. Use the BIOS returned value in dl to access the drive. Don't assume dl=0.
2. Use int 13h, ah=8 - Get Drive Parameters to obtain the emulated geometry.
   Don't assume SecsPerTrack=18 or Sides=2.

> Thanks for the info, but I'm confused! Earlier you said:
>
>> On a side note, MikeOS will only boot and run with a USB flash drive on
>> older computers which support USB floppy disk emulation, as well as on
>> some older computers which support USB floppy disk emulation with a small
>> capacity flash drive under 1Gb.

Presently MikeOS _assumes_ floppy disk geometry and dl=0.
This prevents boot and run on PC which emulate the flash drive as a hard 
drive.

> So will USB flash drives with the floppy disk image injected work on
> any PC, or only older ones that support USB floppy disk emulation?

It will work on any PC simply by following the guidelines.
Remember, historically FAT12 was a valid hard drive format. It is
still a valid USB flash drive emulated hard drive format.
With these guidelines the image will boot and run under either emulation,
and of course will also boot and run from a real or virtual (loopback) 
floppy drive.
