On Thu, Sep 8, 2011 at 6:34 AM, Blacklight <blacklight at cordilon.net> wrote:
>
> Just wondering if anyone would be interested if I were to design and
> implement a replacement for cli.asm. The current one is limited in its
> abilities and effectively forces the user to use a plethora of menu-driven
> tools to accomplish what he or she wishes to do.

That could be good, providing it retains the MikeOS simplicity and
clarity. The ability to do some file operations (DEL, RENAME etc.) at
the CLI would be a bonus.

Implementing command line parameter passing to programs would get a
bit more complicated, but I might add that myself. I think I'd use
os_string_parse and make a new system call -- os_get_cli_params or
similar. AX, BX, CX and DX would then point to zero-terminated strings
for four parameters (I doubt anyone would need more in MikeOS!)
