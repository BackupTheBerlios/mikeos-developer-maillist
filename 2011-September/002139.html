4.1.1 was unstable because it was built with optimisations enabled.
Previous versions may have been stable out-the-box, but become
unstable if compiled with newer versions of NASM (the build scripts
didn't disable optimisation).

So it's always a good idea to use 4.1.2! I'll try to see exactly what
the problem is, in the next few days. The kernel built with
optimisations is only a few bytes smaller than without, so it's
probably just an instruction or two where we have to explicitly state
the size of an operation. Perhaps...!
