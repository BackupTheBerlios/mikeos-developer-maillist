Give it a go with Matej's tokenize routine (thanks Matej!) and see how
you get on. If you have problems, I'll take a look. I'm not sure what
the best approach is yet -- apps need some way to find the command
line string in RAM and parse it.

They also need a way to get a parameter count, so that programs
launched via the menu know that there aren't any extra params to
check. In Unix style, there's always at least one parameter/argument
which is the program's executable.
