Hi folks,

Here it is: http://prdownload.berlios.de/mikeos/mikeos-4.2b1.zip

Full changelog below. All testing would be muchly appreciated! NOTE:
the original EDIT.BIN is on the disk, and Troy's modified version is
TROYEDIT.BIN.

This is because there appears to be something funny happening with
Troy's version -- try opening MEMEDIT.BAS or CALC.BAS in it and
hitting F8 to execute BASIC. Then quit the BASIC program, and the
system locks up back in the editor. Troy: please look into this, as it
works OK with the normal EDIT.BIN. I think I've traced the culprit to
somewhere in the "setup_screen" routine and all the string joins it
does -- does it need to do those every refresh?

Anyway, thanks in advance for any testing reports folks!

Mike


# New program: Memory Manipulator (MEMEDIT.BAS) from Joshua Beck: a
colourful program for editing bytes in memory, written in BASIC

# New program: ASCII Artist (DRAW.BAS) from Joshua Beck: lets you
draw, load and save images created with ASCII characters

# New program: MikeTron Reloaded (MIKETRON.BAS) from Joshua Beck: new
features, bonuses and explosion effects

# New program: Calculator (CALC.BAS) from Joshua Beck

# Updated CLI from Troy Martin: can now pass parameters to programs.
The tokenised (elements separated by zeros) CLI string is passed in
SI, and the number of arguments in DX. Plus: new file management
routines to DEL, COPY and REN files

# Updated EDIT program from Troy Martin: takes filename at CLI prompt,
plus new keybindings to create a new file and save with different name

# New system call: os_string_tokenize from Matej Horvat -- reads
tokens separated by specific char from a string, returning pointer to
next token

# New system call: os_string_compare_casei from Troy Martin -- like
os_string_compare, but case-insensitive

# Bumped API to version 14

# TEST.PCX file (in programs directory) renamed SAMPLE.PCX

# Bugfix: os_get_time_string vector is 0054h, not 0055h

# Bugfix: can no longer call BASIC in EDIT.BIN when file is empty

# Handbook updates
