>How would CALC.BAS access the FPU?

In general terms using the FPU as shown below.

Obviously MikeOS does not support it natively
but could be done via POKEs.

Also see the attached pic.

;============================================
; Simple x87 math co-processor test.
; Compile with A86 then run with DEBUG
;
; var_one contains the number 65535 (FFFFh)
; var_two contains the number 2
;
; once run under DEBUG with g=100
; the answer var_three is at cs:011Bh
; which should be 01000100h hex not bin (65537)
; keep in mind that each var is 4 bytes
;
;============================================

finit

fld DWORD var_one
fld DWORD var_two

faddp  ; add the top copies (and pop)

fstp DWORD var_three ; store and pop

fwait

int 3h

var_one DD 65535
var_two DD 2
var_three DD 0
