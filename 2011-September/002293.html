Once you have the answer in the memory offset (4 bytes),
you parse each of the 4 bytes, convert and show it as ASCII
using the "NUMBOX" function.

as the 4 bytes.
I know that each variable ($1 to $4) is 16 bits, but it may make
it easier if you use each one as a byte, representing each
of the 4 bytes you get as the answer.

What is the current limit (number of chars) of "NUMBOX" ?
It would require it to handle 10 characters max as
2 to the power 32 is 4294967296.
