I always use 256 colour RLL encoded Zsoft PCX format.
Apparently there are differences in the several versions of PCX that Zsoft
came out with.
Open your PCX with a hex editor and check the following bytes in the header:

Offset 00h = 0Ah = Zsoft PCX
Offset 01h = 05h = version, it should be version 3 (yes 05h = version 3 )
Offset 02h = 01h = Run Length Encoded
Offset 03h = 08h = number of bits to represent a pixel (per plane)

Then at Offset 41h there must be a 01h which means the pcx only has 1 plane.

I have stuck to these "rules" and never had a problem.
