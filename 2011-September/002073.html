>  - It seems that there's no way of exposing the full version string of
> MikeOS to a program. This isn't a critical issue, but it would be useful for
> programs that are dependent on, say, the space between the end of the kernel
> and the disk buffer.

I'd rather programs stayed within the 32K allocated to them. Using the
extra space is a neat hack, but it could break at any time so that's
not something I'd like to encourage by adding official ways to access
it etc. If you need more RAM, you could always start playing around
with extra segments :-)
