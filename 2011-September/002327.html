Hi all,

Yesterday I was messing about with ser2net and netcat
on my Linux box, to "bind" a TCP/IP port to the
serial port for data logging devices a customer has
installed at several sites.

Then I had an idea, since the Linux box takes care
of all the "nasty" low level TCP/IP stuff and MikeOS
already has support for serial ports, why not connect
a MikeOS machine with a serial port thru a nix box
and out onto the web.
Should be rather interesting.

I'm not for a minute suggesting a web browser for
MikeOS but it would be fun and educational to make
a sort of 'wget' app for MikeOS.

The HTTP protocol (on top of TCP/IP) is quite easy to
implement, for example the MikeOS box could send
out the following text:

GET /pipermail/mikeos-developer/ HTTP/1.0
Accept: image/gif, image/jpeg, image/png, */*
Accept-Language: en-us
User-Agent: Put anything you want here :-)
Host: lists.berlios.de
Connection: Close
<new line>

This request would go out as plain text via
the serial port to the nix box which in turn
would send it out to the web.
Once the remote server responds, it's also
in text which would be fed back to the MikeOS box:

HTTP/1.0 200 OK
Date: Fri, 28 Sep 2011 23:59:59 GMT
Content-Type: text/html
Content-Length: 886
<new line>

Followed by the html content which
could be displayed on the screen or
even saved to disk.
Obviously the normal 64KB restriction
still applies but it would be fun to see
exactly what is achievable.

Paulo.
