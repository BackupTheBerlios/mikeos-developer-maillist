APPDEV version 2.1 is ready to use.
I fixed up everything in that buggy 2.0 release (except LOADBOX).
This time I tested my final version.
Once again read the manual and examples before using.

The changes are:
---ANIMATIONS--- (ANIOPEN, ANICLOSE, ANISTART, ANISTOP, BLOCKBOX,
BLOCKBOR, BLOCKSCR)
When every you open a box, close a box, start the program or end the
program it shows window animations.
It's no compiz but they make boxes look good with no extra effort on your part.
At the start of a program you can either set $y to "ENABLE" or
"DISABLE". If nothing it will be assumed enabled (so older programs
will automatically have animations.
They consist of several drawing phases 1/10 of a second apart. Note
these may be too slow on older machines (anything less than a Pentium
I wold recommend disabling it).
---BOX REDRAW--- (BOXSAVE, BOXREST)
A new method of saving the stuff below the box and restoring it after
is now in use rather than having to refresh the whole screen. CONTENT
is now only called by STARTPRG on 'FULLINT'.
This is important because the screen stuff is likely to be
irreplaceable. Once again this is integrated into box functions and
will automatically work once you replace the framework in you program
with the new. It require no further coding from you.

If you have written an application in a previous version just copy and
paste everything from "rem appdev content" over the older version of
it in your application using your second favorite text editor ;-) .
Remember to backup and custom functions or changes you have made.

Good luck in writing applications!
