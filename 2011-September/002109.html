Also a couple of notes for anyone planning contributions to MikeOS 4.2:

* Please keep to the coding style (tabs for indentation, aligned
comments etc.) I know it's not a big deal, but as our project is a
learning OS, I want it to be as clear and consistent as possible.

* If you change anything in existing code, please document it! I'd
like doc/CHANGES.TXT to be just as useful as the other documentation,
so if you fix a tiny bug somewhere or update something, please make a
note, however small, so that we all know what's going on.

Thanks!
