I'm not sure I fully understand what you're considering here, but I'll try
and explain what MikeOS does (I'm sure Mike will dive in and correct me
somewhere!).

As I understand it MikeOS has a list of syscalls linked to static locations
in the memory (hence the need for a memory map). You then reference the
syscalls by name from your kernel or apps (you can reference them directly
from memory, but by referencing names only you can alter the memory map
without rewriting most of the kernel!). Essentially the names are like
aliases to the various memory locations which store the
value/interrupt/instruction you may want/need.

I will confess at this point though that I'm a C programmer and fairly new
to ASM, but if you have a great idea in your head and time to make it
reality then you can get most things to work. You're idea looks interesting
and I'd be interested to see your code :-)
