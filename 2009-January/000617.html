Yep, simplicity is the overriding goal, but I still want to have a
good API to encourage more external program development. MikeOS is
working in a limited environment with only 64K to play around with (as
I want to avoid segment register horrors), but given that the kernel
is only around 7K right now, there's loads of space for more system
calls.

So please feel free to keep submitting code -- if it's right for
MikeOS I'll use it, or I'll take it as inspiration for more ideas! My
goal has been to replicate as many features of a typical C library as
possible (within the goals and constraints of MikeOS), thereby making
it easier to write apps. So more system calls are definitely welcome
:-)

You mentioned working on a kernel -- how's that going? I assume it's
