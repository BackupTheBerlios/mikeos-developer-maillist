I think I may have cracked it - you can actually compile asm using gcc.
Usually you would do it inline, with C, but as this link shows, it's
possible to use gcc rather than NASM to compile intel-syntax ASM:

http://asm.sourceforge.net/articles/linasm.html#Compiling

This means that the build script only needs minor changes, and that ApolloOS
may no longer require NASM as a prequisite. :-)

It is also possible to compile 16-bit binaries using gcc, as shown here:

http://devpit.org/wiki/Compiler_Options_for_Creating_Odd_Binaries

So a MikeOS build script port does not seem entirely impossible :-)
