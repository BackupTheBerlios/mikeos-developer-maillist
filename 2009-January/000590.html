Also, on that note, the current os_get_random system call is rather
primitive; it simply uses the BIOS clock routines to generate a number
between 0 and 255.

It's fine for the hangman game, but for a program which needs a
succession of random numbers generated very quickly, patterns will
emerge.

So I would like to turn it into a more versatile random number
generator which can, say, take a minimum in AX and maximum in BX, and
generate a random integer between the two. Any list lurkers are
welcome to take it on!
