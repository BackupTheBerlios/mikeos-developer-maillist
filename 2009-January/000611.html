Great! More goodies for 3.2 :-)

>        - Disk image is ALWAYS created from blank floppy image.

I'm not sure if we should do that, though. A user may add extra
programs / images / text files to the disk for testing, and these
would be erased each time MikeOS is built. So the user would then need
to modify the script to add these extra files each time.

I think it's simpler if we just check for disk_images/mikeos.flp, and
if it doesn't exist, create a blank floppy (as in build-linux.sh).
