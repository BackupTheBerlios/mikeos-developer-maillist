A few months back, rudimentary graphics capability was pulled from 
MikeBASIC because we had, quite frankly, no use for it without the 
ability to draw arbitrary lines. There was also a statement regarding an 
internet high-five to the first person to get that working in MikeBASIC.

Attached is my implementation of Bresenham's line algorithm in assembly 
language, and a demo that draws three lines in different directions with 
different colours. It's probably still not enough to get graphics back 
into MikeBASIC (we still would need more primitives for that), but it's 
a start.

The code was hand-translated from a sample implementation in C at 
http://www.edepot.com/linebresenham.html. My code definitely needs to be 
simplified (currently, it's almost a direct translation of the C code, 
variables and all), but y'know. It works.

Cheers,
Troy
