I haven't decided on the specifics yet, but the functions it will have will 
mostly pertain to window control placement (buttons, checkboxes, things of 
that nature) and it will also have functions that will provide a more 
flexible and easier way to access the disk and load files. While it will be 
possible to make direct MikeOS API calls while in the Xypher environment, 
any existing MikeOS application will not run properly without a few 
modifications. I plan on designing my own executable format based on the 
MS-DOS MZ format.

At this point, I'm still drawing out the memory map, API functions, etc. and 
I'm working on some primitive screen functions.
