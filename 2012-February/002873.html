I seriously thought about doing it in mode 13h, but when I drew out the UI, I realized that there really isn’t a lot of space left to work with. Seeing that 320x200 is almost as half as big as the resolution Windows 1.01 used is kinda pathetic, LOL.

I wanted to compromise between mode 13h’s vast color palette and the 16 color modes’ larger resolution. Since most if not all (I can’t remember) of the 16 color modes used planar memory, I decided to opt with the largest resolution of them all: 640x480.

Also, I kinda wanted a challenge and not just some easy to program video mode. I looked closer at it this morning and it doesn’t seem that hard, actually. App developers won’t have to worry about messing with the VGA registers. That’ll all be handled through the API. :-)

Do you really need that many pixels?
As far as I can tell VGA mode 12h is 640x480x16 and you have to mess around with "planes" and stuff like that to use it.
I think a far better choice would be mode 13h. 
This mode is 320x200x256, and has a single byte of memory in the video segment for each pixel It's really easy to use, you could have 
your video driver written in ten minutes.
