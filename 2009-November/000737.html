Hi,

> Great job, but just another little question : why is the basic interpreter
> inside the kernel and not an external program ?

Good question. Originally it was a standalone program, but that
started to make things complicated: MikeOS had to load BASIC, which
had to then load the .BAS program. If you wanted to run code directly
in the editor (EDIT.BIN), you'd have:

1 - MikeOS loads EDIT.BIN
2 - EDIT.BIN loads .BAS file
3 - EDIT.BIN has to load BASIC somewhere else in RAM

So it was easier to have os_run_basic as a kernel routine. Also, this
frees up more RAM in the program part of the MikeOS segment -- the
second 32K.
