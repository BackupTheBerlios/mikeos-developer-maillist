Lo,

Yes, I can confirm that MikeOS boots from USB keys. Gene helpfully
investigated this with the Flashnul tool, and I've found out that you
can do it using the normal Unix tools (well, Linux so far).

1) Insert USB key
2) If the drive auto-mounts, unmount it
3) Enter 'dmesg' and take note of the device name given to the key you
inserted (eg /dev/sdc -- ignore numbers at the end)
4) As root, run 'dd if=disk_images/mikeos.flp of=/dev/sdc' (replacing
'sdc' with the device name)

Remove the USB key and you should be able to boot from it. NOTE: If
you plug the key into another OS, you can access files on it -- eg
retrieve a BASIC program you've written. However, it may also alter
the structure of the key so that it can no longer properly boot
MikeOS, in which case you'll have to repeat the above steps.

I'm going to add all this to the User Handbook, along with information
that Gene sent me for using Flashnul on Windows. This is great stuff
-- I've been running MikeOS off a flash key on a laptop for a few
hours now, testing BASIC stuff, and it works brilliantly.

Perhaps it's time for some MikeOS-branded USB keys :-)
