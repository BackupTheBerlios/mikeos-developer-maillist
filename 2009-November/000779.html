The '\n' notation us just a C-like way of representing newline
characters -- there's no actual slash or 'n' stored in the text file.

What you do have, however, is two single-byte characters which
represent a linefeed and a carriage return. Unix-type text files use
the first, and MS-DOS like use both. For instance, take a text file
containing these two lines and do a hex dump:

hello
there

Between the 'o' of 'hello' and 't' of 'there', you'll see a byte with
the value 10 (decimal) aka 0A (hex). That's the linefeed character,
and on Unix systems (and in the MikeOS editor) that is sufficient to
generate newlines in text.

On DOS/Windows boxes you also have 13 (decimal) aka 0D (hex), which is
a carriage return. However, for files created in the MikeOS text
editor you only have to deal with linefeed (10, 0Ah).

In summary: Unix/MikeOS text files are streams of ASCII bytes with 10
(decimal) / 0A (hexadecimal) representing newlines. If you want to
convert a DOS-like text file to a Unix style on a Linux box, install
'tofrodos' and run 'dos2unix' on the file in question.

Hope that explains everything!
