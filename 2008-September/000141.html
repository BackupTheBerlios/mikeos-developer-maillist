Well... Mike wrote the answer, I write another one.

GCC is very good compiler, but this should be used for 32/64bit environment.
The option ".code16gcc" makes GCC to generate x86/16bit binary,
but the binary is suitable for 16bit real-mode -> 32bit protected-mode
bridge such as bootloader for 32bit OS.

And GNU's toolchain is very flexible. But we have to remember that
the flexibility sometimes make things difficult.

I think GCC is too complex to use MikeOS, so I choose Bruce's bcc.
