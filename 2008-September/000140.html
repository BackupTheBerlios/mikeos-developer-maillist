> Great work on the updates. Given the license differences, for
> simplicity, it might be better to have MikeOS in one package and the C
> files in another -- then there won't be any complications. What do you
> think?

Thank you for good idea and I am planning the licence of libmikeos' package
will be LGPL and MikeOS as follows.

	uaa at framboise:~/libmikeos$ ls -l
	total 28
	drwxr-xr-x   2 uaa  users  2560 Sep 21 20:08 build/	-> LGPL
	drwxr-xr-x   2 uaa  users   512 Sep 14 17:59 include/	-> LGPL
	drwxr-xr-x   2 uaa  users   512 Sep 14 21:28 lib/	-> LGPL
	drwxr-xr-x  12 uaa  users   512 Sep 20 22:19 sample/	-> MikeOS
	drwxr-xr-x   2 uaa  users  2560 Sep 21 11:42 src/	-> LGPL
	uaa at framboise:~/libmikeos$

Even if almost all of sources in src/ directory are written from
scratch, I think using LGPL is better choice now. 
Because the fundamentals for bcc is under LGPL, so using same licence will
be easy to port bcc's stuffs to MikeOS.

For libmikeos' sample codes, there is no reason to use GPL.
So samples are under MikeOS' license.


Now I am testing int/long-int operation, please wait for next release...
