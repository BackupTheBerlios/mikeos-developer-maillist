Hi all,

I've made the first development release of what will be MikeOS 2.0.
This includes:

1) Tomasz Gorol's hugely thorough mouse driver
2) The current FAT12 write implementation (as programs/file.asm)
3) os_string_strincmp system call (as discussed earlier today)
4) os_int_to_string fixed where AX = 0 (as per Takayoshi's bug report)

If anyone can test the mouse driver on virtual and real hardware, that
would be great. So far I've tried the disk image on QEMU (a bit buggy)
and VirtualBox (seems OK); I'll give proper hardware a go tomorrow.
You can download the whole lot at:

http://prdownload.berlios.de/mikeos/mikeos-1.9.1.tar.gz

Full ChangeLog below. Thanks in advance for testing!

Cheers,
Mike


1.9.1 (26 September 2008)
-------------------------

# Complete mouse implementation from Tomasz Gorol. Adds:

    source/os_mouse.asm
    source/os_mouse_syscalls.asm
    source/macro.mac

  Plus it adds an interrupt handler in os_main.asm and makes some
  small fixes to other places in the kernel. Also adds a 'mouse'
  subdir to the 'doc' directory

# programs/file.asm demonstrates the mostly-complete FAT12 writing
  implementation. Still bugs and limitations. Run the program to
  generate MEGAFILE.BIN which is a small program that displays just
  the letter 'H'

# New system call: os_string_strincmp, like os_string_compare but
  limited to the number of characters passed in CL

# os_int_to_string fixed -- previously it wouldn't return anything
  if AX = 0. Now it sets string as '0'.

# In programs/mikedev.inc, os_string_uppercase and os_string_lowercase
  were marked incorrectly as using SI rather than AX -- fixed

# Un-needed commented-out lines removed from programs/keyboard.asm

# In source/bootload.asm, comments referred to first two bytes of
  FAT; really means first two clusters -- fixed
