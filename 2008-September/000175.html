On 9/28/08, Troy Martin <blasterman95 at gmail.com> wrote:
>
> It's edlin-like (with even less features, yuck) but it's still a text
> editor!

Well, you can't edit your text yet -- it's more like 'echo blah >
foo.txt'. But I'm just being pedantic :-)

Seriously, this is a good start. I'm going to release 1.9.2 with the
FAT12 write bits folded into syscalls.asm, so can strip them out of
your program and keep the code neat. What's your plan next? Being able
to edit the lines in RAM would be great, and you've got the various
MikeOS screen manipulation routines to help out (moving the cursor,
clearing the screen etc.)

Or you could take an easier approach for now and work on it as a
command-driven editor -- eg you enter 'EDIT 1' and then you're
prompted for text, 'DELETE 15' to delete a line, 'LIST' to show the
current text, 'SAVE' and so forth. I'd forget about the 512-byte limit
for now -- dedicate the upper half of your 32K app space to the file
data, then work with managing that memory (deleting lines, inserting
lines etc.). That's the trickiest part -- saving is simple and can
come later when I've finished the FAT12 write bits.

So, a very simple but good start. Keep at it! Oh, and thanks for the
label stuff. I should enable warnings in NASM on Linux...
