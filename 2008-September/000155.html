Oh, and as Brendan says on the OSDev forums, you're going about it the
wrong way -- you need a generic parser routine first. Stripping six
characters and cutting out double quotes only handles PRINT for one
scenario; it's completely broken when you have stuff like this:

PRINT "Some text"; 2000
IF A = 1 THEN PRINT "Foobar"

PRINT can be used in many different scenarios, as can all BASIC
commands, hence why you need a generic parser routine that determines
the command, number of parameters and whether they're valid. It also
needs to be able to put the command in a context, as in the IF line
above.

Read up on language parsing (and learn a lot more x86) and you'll have
a better idea of what's involved -- just jumping in and redirecting
PRINT to os_print_string gives you, well, nothing to handle loops,
IFs, multiple parameters etc. and you're stuck permanently at stage 1.
