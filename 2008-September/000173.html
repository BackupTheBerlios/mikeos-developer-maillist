It's edlin-like (with even less features, yuck) but it's still a text
editor! It's basically a modified file.asm (same routines, added in the
interface myself) so it can only save up to 511 bytes (yes Mike, it's 511,
something's not saving the last byte, but it will save the last null byte if
the file is supposed to be 511 bytes O.o)

It's a great start, though. All keyboard-produced ASCII characters work,
save enter (have a handler, but it's broken) and backspace (haven't written 
the
handler yet.)

But it's good enough to release! It's stable too!

PS: Mike, your file.asm also had missing colons on labels (lines 523, 571,
620, and 669) :-) I fixed them, even though they didn't affect assembling
and running.
