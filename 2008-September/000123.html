The keyboard driver is still messed up. It's my next project. It was recently labeled stroke-induced by someone at the OSDev forums for the lack of a jump table and the continuous re-usage of "printing character, moving cursor" in EVERY CHARACTER SECTION (wait until you see the source for it, you'll die in horror.)

WRITTEN TODAY:

The 4GB flat memory model is going to be fun to work with. Not that TBOS will ever grow to need all that though. :P

My continous re-usage stated above is no more, since doing a "call .printandmove" is much more efficient. Let's see what medical condition induced this one...

I think the next TBOS core project will be a shell and some more syscalls. My goal is to work on a FAT12 read driver and toss in syscall vectors in between revisions and have FAT12 ready by 0.2.10. Version scheme below:

Beta versions: 0.2.xx
Release versions: 2.x.x
First 'x' is major revision number, second 'x' is minor revision number.
Current version is 0.2.01.
