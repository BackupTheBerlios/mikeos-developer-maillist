Whee! It's almost running! All it needs is bugfixes. On that note, is there a way to redefine labels (for a db) in assembly?

Here's what will be in 0.0.1:
- puts();  strange things will happen if you don't add the semicolon...
- return 0;  this just sends a ret to the intermediate assembly file (temp.asm) and does the compiling with nasm.

Most things that need a semicolon at the end in normal C need them in this C in order to make them translated and added to the .asm, so check your code before hitting enter!
