I've got an include file called stdio.inc that defines a puts macro that 
makes a zero-terminated string from the parameter, mov's the location into 
SI, and calls os_print_string. So it prints out the string but seems to hang 
on the ret.

On the C64 port note, it now does more than print a few messages. It has a 
few commands from MikeOS-x86, like ver, help, prompt, and cls. It can also 
list line 1000 and go back to the MikeOS CLI, and execute a program at line 
1000. Of course it can also drop you back to BASIC so you can write a 
program starting at line 1000 and then run it from basic or MikeOS. wherever 
you run it from, of course, it will send you back to MikeOS if you have GOTO 
190 at the end, which you should. Although you should always run it from 
MikeOS.
