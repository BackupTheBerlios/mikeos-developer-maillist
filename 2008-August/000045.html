Hmm... I've tried it on a few physical devices.

On an Eee PC 900 (USB floppy drive) - Most of the OS works but you can get sluggish performance. The 'mouse.bin' module loads but then terminates straight away (you don't get the chance to see if the touchpad works) - perhaps this is a debugging measure, as it wasn't designed for trackpad mice!

Likewise a similar thing happens on an old Toshiba satellite (You know the old grey ones with the green sponge in the middle of the keyboard as a mouse?). The mouse.bin module mostly works, but I do sometimes get some bizarre co-ordinate addresses in the bottom-left.

It seems to work perfectly with my desktop machine though (with a PS/2 mouse). though USB  mouse performance tends to be a bit more erratic...!

  Lo all,

  Tomasz Gorol has contributed a complete mouse driver for MikeOS.
  Here's the patch against 1.4.2 (it's a whopper!):

  http://mikeos.berlios.de/mouse/mouse.txt

  This implements native system calls, some DOS compatibility, and an
  example program. I've uploaded disk images (floppy and CD-ROM) here,
  with the build name 1.4.2M:

  http://mikeos.berlios.de/mouse/disk_images.zip

  So far we've tested it in QEMU and VirtualBox -- I'm going to try some
  real hardware tonight. Any other testing muchly appreciated!
