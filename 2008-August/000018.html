That really does sound good - once you have networking between MikeOS
machines, what's to stop someone building a simple text-based chat client to
use on the interface? The next step would be of course to get some
compatibility with TCP/IP to allow internet access, but I think we're a
while off that!

I'm presently messing about trying to get MikeOS's bootloader to recognise
my 32-bin kernel image (I'm trying to make a 32-bit MikeOS (rewritten in C)
which is very difficult! I think the desktop GUI I'm offering people could
be a while off!).

That sounds like rewriting the whole OS (and to an extent it is), but MikeOS
app code should still work - I'm working on a 'MikeOS-compatible' API which
will allow you to port MikeOS app code to ApolloOS without changes (you will
need to use the relevant build script bundled with Apollo source code, as
pre-built MikeOS binaries won't work natively - I'm essentially using the
same API references you use in your code but re-routing it to ApolloOS
kernel instructions).
To get the most out of ApolloOS however you would need to use the native API
(in 32-bit) which contains an API and useful header files - and have the
full benefit of being able to use 4GB RAM!
