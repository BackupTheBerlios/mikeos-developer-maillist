It won't work because program execution begins right at the beginning
of the file (ie the start label, after the directives, which don't
assemble to actual code).

So, MikeOS loads the resulting binary and starts executing it. The
first few bytes in the program are

  'Hello...'

etc., which naturally aren't valid machine code operands, and hence
why the program appears to do nothing (it could just as well crash).
The program needs to start with machine code instructions -- data
shouldn't be in the same place as instructions otherwise all hell will
break loose!

Any non-instruction stuff (db, dw etc.) needs to be separate from the
machine code, eg at the end.
