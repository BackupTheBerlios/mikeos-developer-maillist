This is something I've found out and written while digging through the 0.0.1 archive of the Linux Kernel.

Linux 0.0.1 boots, does the unpleasant bits of booting, sets up a 32KB app space, and loads the kernel.
MikeOS 1.4.2 boots, does the unpleasant bits, and loads the kernel, which has a ** 32 KB app space! **

There's some comparison here.. I can feel it.

Now a few differences:

Linux is in AT&T syntax asm and C, MikeOS is in NASM syntax assembly.
Linux was concieved to make a better Minix, MikeOS is a "learning tool."
Linux is a mainstream, huge OS. MikeOS isn't... yet... muHAhaHAHaHaHaAA!!!
Linux is meant to be a general-purpose OS, MikeOS is meant for world domination.
Linux uses interrupts for functions, MikeOS uses syscalls.
Linux only supports Linux executables, MikeOS supports MikeOS and DOS executables.
Linux rules, MikeOS rules even more!
