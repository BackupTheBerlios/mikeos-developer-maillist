Paulo makes good points there for those basing their OSes on MikeOS
code. Good to have this stuff in the mailing list archive.

I'll also add: MikeOS system calls are not always safe for
multitasking. Some of them (eg os_int_to_string) pass back a pointer
to an internal buffer which the calling program then manipulates.

With co-operating multitasking (as I guess pre-emptive is out of the
question here), when you yield back to the OS, you cannot guarantee
that a system call's internal buffer is in the same state when you
return.

Something to be aware of, for those using code from MikeOS system calls!
