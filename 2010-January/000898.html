Regarding the whole 16/32 bit issue, Bazza has a point that MikeOS is
currently 386 minimum and therefore won't work on some older 16 bit
machines.

However, as MikeOS is a learning tool more than anything else, I've
been happy to stick with 386 as a minimum. It keeps things simple.

Double however, if Bazza finds that only a handful of small things
need changing in the source code for it to work on pre-386 boxes, and
it doesn't add any complexity to the code, I'm happy to make the
changes. (Although I wouldn't want 'pusha' being replaced with a
billion 'push ax', 'push bx' etc. instructions :-) )

If 8086/80186 compatibility makes fundamental changes to MikeOS, but
Bazza makes a version that works on those machines, I'll still put it
up in the Downloads section -- it'd still be great to have.
