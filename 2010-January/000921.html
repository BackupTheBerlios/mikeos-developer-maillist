Seriously though, I agree with you Mike that the more projects there
are for MikeOS the better.
For example, myself and Bazza were discussing a few things off line yesterday
and he posed a very interesting question, how does one get real world data in and out
of a new netbook/laptop which has only USB with MikeOS?

I've never really given this much thought as normally I would just use USB or even Ethernet
and problem solved, but on MikeOS this would be unfeasible.

So after some head scratching I think I've come up with a solution....IrDA!!
For those who don't know, Irda is a bi-directional infra-red port found on many laptops, netbooks
and smartphones.
On PC's, it can be enabled in BIOS by mapping COM2 to the IrDA port.

IrDA uses some strange protocols but only when taking advantage of the maximum speed which is 4MB/sec.
At lower speeds, 2400 to 115K baud, we can dispense with all the protocol overhead and simply 
transmit and receive raw bytes very much like when using the serial port.
The IrDA port uses a slightly different physical layer to RS232 (serial ports) but should be easy to implement
using very basic (no pun intended) interface circuitry and I suspect many of the serial port routines can be re-used for the IrDA port.

So I have decided to look further into this and make a IrDA interface for MikeOS.
Will keep you posted.

Paulo.
