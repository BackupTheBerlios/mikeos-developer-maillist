Hi,

It's all on the net: http://faydoc.tripod.com/cpu/ret.htm

In short, when you use the "call" instruction the IP register
(location of the code being executed) is pushed onto the stack. When
you use the "ret" instruction it pops the data from the stack and puts
it into IP.

If you want to retrieve the data without using "ret", you could use
"pop" to get it off the stack. Or use "mov ax, [sp]" or similar (I've
never used the latter but some kind of variant must exist).
