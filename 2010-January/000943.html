Bazza,

Attached is a possible circuit to interface your serial ADC to the keyboard.
The circuit uses the CAPS LOCK and NUM LOCK to control it.

The CAPS LOCK is used as a clock whilst the NUM LOCK forms a chip select
for the ADC and also a gating control to only enable the data coming out of the ADC
whilst you are taking a reading.
If this was not done, you would soon have a lot of key presses in the KB buffer.
The "AND" gates steer the data output from the ADC to either the "1" key when the ADC output is high
or the "0" key when it's output is low.

This would make your software easier to write as you would for example get back from the ADC a string like such:

11000101

I've kept the circuit as simple as possible and it only uses 1 extra cheap and easy to get IC, namely the 40106B.
If you need separate controls for the ADC chip select and the "gating" signal, then keep NUM LOCK as the chip select
and use SCROLL LOCK for the "gating" control.

Have fun,
Paulo.
