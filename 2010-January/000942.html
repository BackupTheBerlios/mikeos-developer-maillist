Hrm, shame! How did it not work? Did it give any error messages?

> IF NOT EXIST c:\mikeos\nasmdos\nasm.exe GOTO NoNasm

The problem with that, is that it requires a specific filesystem
location for the binary. With the original script:

> nasm 2> nul
> if ERRORLEVEL 9009 set MIKEOS_NONASM=1

It runs NASM anywhere in the PATH, so it doesn't need a specific
location. It redirects error messages to nul (ie doesn't display
them), using 2> like in Unix. If the previous command cannot be found,
the ERRORLEVEL is set by the shell to be 9009.

Now, this all works fine in Windows 7, but I gather than Win7's shell
is much better and more Unix like than in previous releases. Still, I
was hoping that XP had support for error message redirection and
ERRORLEVELs!
