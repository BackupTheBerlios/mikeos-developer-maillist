Hi Mike,

The new build.bat for windows did not work for me.
My files are in different directories, but the bat file still did not work
even after updating the paths.

I'm using XP PRO SP2
Below is the modified bat file I made which does work.
Obviously change the paths as required and take out the rem statements
to actually call the programs together with the command line swiches.

Paulo.

==========================================================

echo off

IF NOT EXIST c:\mikeos\nasmdos\nasm.exe GOTO NoNasm
IF NOT EXIST c:\mikeos\imdisk\imdisk.exe GOTO NoImdisk
IF NOT EXIST c:\mikeos\partcopy\partcopy.exe GOTO NoPartcopy

echo If you see this Nasm exists
rem nasm would be called here

echo If you see this Imdisk exists
rem imdisk would be called here

echo If you see this Partcopy exists
rem partcopy would be called here

goto end

:NoNasm
echo NASM not found: please install from http://www.nasm.us
goto end

:NoImdisk
echo IMDISK not found: please install from http://www.ltr-data.se/files/imdiskinst.exe
goto end

:NoPartcopy
echo Imdisk not found: please install from http://www.virtualobjectives.com.au/utilitiesprogs/partcopy.htm

:end

==================================================================

Hi all,

Here's an updated version of the Windows build script (BUILDWIN.BAT)
that checks for the necessary tools (NASM, PARTCOPY, IMDISK) and
prints an error if it can't find any of them. Previously there weren't
any checks so it just clogged up the screen with error output if you
were missing anything.

We've tested this on Windows 7 -- if someone could give it a try on
other Windows versions that'd be great! Rename it .BAT  -- it's .whizz
here to stop Gmail complaining.
