Hi Scott,

> 1. Boot into MikeOS via PXE (is this possible?)

I don't know much about PXE, but if it's possible to send a floppy
disk image over the network to a machine, put it somewhere in RAM and
load it like a virtual floppy, that's doable!

> 2. "dd" or some other mechanism to transfer complete bits of a
> filesystem to the hard drive (over the network)

MikeOS has no concept of hard drives, only floppy drives, so something
like that wouldn't be possible without a lot of recoding.

> 3. Boot the first sector of the hard drive (withOUT "rebooting")

Well, via MikeOS you could load a bit of code into somewhere in RAM,
jump to that code, and then have it load the first sector from the
hard drive into the usual place (7C00h) and jump to it. You can read
hard drive sectors using BIOS calls so you could do it with a bit of
assembly code.
