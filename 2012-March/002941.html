I'm converting 4.3 to MikeOS32 and noticed a couple of things in basic.asm:

line 980
 inc word [loop_in]
 inc word [loop_in]
line 1890
 dec word [loop_in]
 dec word [loop_in]
- loop_in is a BYTE variable. Since it's used in the code as a WORD index
  it probably should be a WORD variable. This would necessitate the changing
  of all loop_in references to WORD.

line 277
 mov di, do_loop_store
 mov cx, 10
 rep stosb
- do_loop_store is 10 WORD variables so CX should be 20. It's only 
programming
  style, but in the clear_ram function at line 261 a CX equal to the number 
of
  WORD variables and a REP STOSW (with AX equal 0) would be a simpler 
presentation.
