I don't think we're all on the same page with what this TUI layer is 
supposed to do.

The application needs to be developed to support the layer. Applications 
that do not need to access the windowing layer don't have to call it, and 
will therefore run as they normally do.

> I still don't see the point of a TUI layer like this. It's a neat 
> exercise, yes, but all console programs so far are really built around an 
> 80x25 console that is printed to and has input captured directly through 
> the BIOS. Windowing console programs like this would be a hell of a lot of 
> work. Unless you have a better solution than hooking every console-related 
> system call, you'd be better off just forking MikeOS for a project of this 
> scale.
