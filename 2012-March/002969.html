I mentioned not too long ago about a GUI layer for MikeOS applications that I’ve been working on called Xypher. After much thought and comparing screen resolution to memory size to difficulty ratios, I’ve decided to, instead of settling with a full blown graphics mode framework with custom font handling and the works, to opt with the simple mode 03h text mode.

The framework is a TSR that will be loaded into a segment out of reach from the application and will be accessed via interrupt 30h. The windows’ designs will be specified in an information block. This block will consist of the unique window ID number, the window type, and the controls it will have. I haven’t thought about it much beyond that.

There won’t be any mouse support for now. Nor movable windows. (However, windows can overlap each other.)

Does anyone have any suggestions or features they would like? I don’t want to really get into this and leave out something important.
