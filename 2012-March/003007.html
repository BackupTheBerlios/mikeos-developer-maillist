On 29.03.2012 19:41, Blake Burgess wrote:
> How does an option for custom text modes like 90x60 or 80x50 sound?

80x50 would use 8x8 square characters (well, 9x8 technically, in some 
instances, but that's more square than 8x16), so it would be easier to 
draw square things, I guess.

I still don't see the point of a TUI layer like this. It's a neat 
exercise, yes, but all console programs so far are really built around 
an 80x25 console that is printed to and has input captured directly 
through the BIOS. Windowing console programs like this would be a hell 
of a lot of work. Unless you have a better solution than hooking every 
console-related system call, you'd be better off just forking MikeOS for 
a project of this scale.

Not trying to be rude or discouraging, just realistic.

Please tell me if I've gotten the whole point of this exercise wrong.
