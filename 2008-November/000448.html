Wow - it's been a while...!

I'm getting into a bit of trouble with a virtual terminal I'm writing in C
(here we go..!). I'm trying to parse a character array to find spaces. Once
I've found the first space I grab all the previous characters and place it
in an array called '*command' and all the rest in '*cmdParameters'. However,
I keep getting an 'lvalue required as left operand of assignment' when
comparing the array to spaces. Here's some code...

void vterm_send()    //this is where all the magic happens. The string is
parsed for spaces (to allow parentheses) then the appropriate function is
run (or an error message if unrecognised)
{
    int i=0, j=0;    //initialize the loop handlers...
    int status=0;    //this defines whether this is the command or a set of
parameters...
    unsigned char *command;        //this array will hold the command
    unsigned char *cmdParams;    //this will hold the parameters - there
will be a local parameter handler in vterm.c
    unsigned char letter;        //temporary character handler used in
loop
    strHandle--;    //we have to do this or we'll be an array element ahead
of ourselves - we can use strHandle as our maximum value
    for(i=0;i<strHandle;i++)
    {
        letter = cmdString[i];
        if(letter == ' ' && status=0)    //if we've found our first space...
        {
            for(j=0;j<(i-1);j++)        //...up until that space...
            {
                command[j] = cmdString[j];    //...we add things to the
command[] string!
            }
            status++;            //now as we've done the first one, update
the status
        }
        else if(status>1)    //we throw everything else into our parameter
string once we have the command!
        {
            for(j=i+1;j<strHandle;j++)
            {
                cmdParams[j] = cmdString[j];
            }
        }
    }
}

I've highlighted in red the parts the compiler claims to be moaning about.
As you can see I've tried swapping the array element out to a variable to
see if that will work! (If it's something as obvious as last time...!!!).
There are no other errors flagged up about the syntax.
