Hi all,

I know that some of you are busy working on your own OS projects, but
I thought I'd write a quick list of tasks for MikeOS 3.0. If you've
been lurking on the list for a while and fancy helping out, take your
pick -- all help is muchly appreciated!

# Test! Get the 2.9.2 release, write the floppy or burn a CD-R, and
test on your machines. I'll make a list for the website of test boxes.
And if you take a photo of MikeOS running on a machine, we can
assemble a nice gallery on the website :-)

# Write a complete, step-by-step guide to building and running MikeOS
on Windows (eg what software you need, using virtual floppy disks
etc). I'll put it in the 3.0 Handbooks.

# Read the Handbook and look for anything that isn't explained well or
which could be written more clearly.

# If you know a bit of x86 assembly language, read through a chunk of
the code and spot areas which are confusing or have no comments --
point them out and I'll fix them!

# Port some small DOS games. Yes, DOS support has been removed in
2.9.2, but many tiny DOS games only use int 21h to exit the program;
if you have the source code to a game, you can just replace that with
'ret' in normal MikeOS style. If the game uses any other DOS calls,
you may be able to switch them to their MikeOS equivalents.

Any other suggestions welcome!
