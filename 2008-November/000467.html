It's a shame to see so many features disappear (I am assuring myself that
TomOS will justify this!), but the changes to the GUI are really very good.
Things look a lot more refined! :)

The only bug I could find was with 'monitor.bin'. If you hit 'r' followed by
enter on the first line you hang the app (and thus the OS) - though I would
imagine this is easily corrected by initializing it or using a cmp statement
(to ensure that the code to be re-run is not in fact a null value!). But
that would be nitpicking!

As I said before, I am looking forward to see what the pair of you produce
with TomOS. (ApolloOS is also coming on leaps and bounds by the way - we now
have Windows development support, and 0.0.7 promises a command line that
handles parentheses and a miriad of bugfixes submitted by contributors (to
the point I've just created a mailing list) - floppy and application support
to follow before version 0.1!)
