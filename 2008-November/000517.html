Hi all,

My name is Paul Sommers from South Africa and have just recently
discovered MikeOS.

I quite like the no bloatware simplicity of it and of course the fact that
I'm not contributing to the "Gates Holiday Fund!"

My interests lie in adapting MikeOS into a sort of embedded system
using old unloved PC's.

These could then be used for simple home automation servers to control
appliances via relays under control of the LPT or serial ports.

I'm an electronics design engineer by profession and have also been
developing
software mostly for the control of hardware under Windows.
The languages I use are Pascal and assembler, both 16 and 32 bit and
Visual Basic for the GUI front-end.

As far as the memory protection is concerned, is it really such a big issue?
Surely the "price" one pays for a nice simple OS like MikeOS is the fact
that
the developer/programmer has extra responsibilities in writing safe code and

not over writing data is one of these.

Surely to properly implement memory protection would mean switching over to
protected mode which is not much fun.

Keep up the good work,
Paul.
