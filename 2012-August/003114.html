Hi Matej,

I really like this -- it's clean, simple, and fits in perfectly with
the MikeOS ethos. Best of all, it can be implemented without impacting
anything else, so I'm up for adding it to 4.4, if nobody has any
objections.

I've been thinking about moving the text editor inside the kernel so
that the bottom 32K of the segment is always available to programs
(instead of the current system, whereby programs can run at 32K or at
32K+(size of EDIT.BIN), which is a bit messy).

On the other hand, the kernel is already around 20K and we don't have
much space left (with the current disk_buffer implementation, the
kernel can only be 24K max). If only there were a solution to all
this... We could try unreal mode, but I don't know how reliable that
is, and AFAIK code segments are still limited to 64K.

Maybe at some point we'll have to use segments (*shudder*). As in, a
64K segment for the kernel, and a 64K segment for programs. I worry
that it'd all become extremely messy though...

Mike
