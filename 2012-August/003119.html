Yotta-1.04x10^24
==============
No new features, many bug fixes, binary attached.
Changes:
  - Page Up/Down no longer mess up the line number
  - Page Up/Down no longer cause the screen & memory to get out of sync.
  - Page Down no longer goes past the end of file
  - Scrolling down no longer goes past the end of file.
  - There is now a DC1 character marking the end of the file
  - Fix a freeze when getting cursor position at the end of the file


>It's already much smoother for editing and scrolling than EDIT.BIN.
That's because it uses a 'smart redraw', meaning it only does a full
redraw when
absolutely necessary. If you insert/delete a character it only shifts
the character
on the current line. Also it has a more efficient rendering routine.

>I used this to
>go to the end of EXAMPLE.BAS, but it seemed to scroll several lines
>past the file's real end point, and then crashed.

I've fixed this problem now, page down will not go to the end and it will no
longer scroll down past the end.

>I think this should replace EDIT.BIN when it's finished

Sure, that may be a while though. I'm hoping to keep it smaller than 10k.
