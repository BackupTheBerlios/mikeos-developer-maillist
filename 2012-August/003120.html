>I've written a memory allocator for use by MikeOS programs
Wow! That's cool.
Can't say I've ever written a MikeOS program that would
need memory management, BASIC or assembly but I
have always been very short of memory, maybe this
could have identify chunks of unused memory for me.

I think load/poke function should check to see if the memory
area is okay first, if not give an allocation error, so it doesn't
overwrite the kernel or a program, I have done this many times.

If we are to multisegment, I suggest keeping a small area at
the start of the user segment available for call vectors,
and change the jumps into far jumps, then add a 'call' to
the start and end of each API call to change the segment,
preserving the original value.

I would be in favour of this, since there are several
half-done applications of mine that could be completed with
more memory.

>Of course, then we'd need a multi-segment memory manager,
>like the one in DOS. And we could have task switching!
You might be interested to know I achieved successful
task switching with version 0.6 of UltraTasker a while back.
Unfortunately it wouldn't switch back. Never figured out why.
