M,
 
> Why the arbitrary limits? Why 512bytes or 1KB for the editor? If
> you're really making this OS for a device that has a hard limit of 5K
> memory, fair enough, but otherwise it just sounds like pain for pain's
> sake!

For the simple fact I want to push it's capabilities as far as possible. It's not so much a project as such - possibly better described as a challenge.
 
I don't know if you have seen - but I took your bootloader and adapted it to a mini-dos like OS with a prompt, where it loads programs of any size (within reason) where the Kernel would normally go. It automatically tries to run 'AUTORUN.BIN'. Any file name passed is automatically given the extention '.BIN'. If a program is not runable it passes back to the prompt. It also offers a 'print_string' and file conversion to the bootloader and any program loaded.
 
Possibly worth looking at for the MikeOS (still offers what the MikeOS bootloader already does). You could then possibly have some programs which run in 'DOS' mode - achieved by jumping back to the bootloader.
