Hi anurag,
>Is there a scope for writing a 'Paint' like program in assembly? I am
>developing my assembly coding skills, but I wanted to know the complexity
>of my problem and what prerequisite knowledge I would need to acheive it.



The complexity depends of how advanced you want your program to be. I'd recommend you have some experience with programming in assembly language and algorithms before attempting this.
 Since MikeOS doesn't have a graphics API, you'll want to create a method of drawing graphics primates such as circles, angled lines and flood fill. you'll need to figure these out for yourself. Try looking for "midpoint circle algorithm", "bresenham's line algorithm" and "flood fill algorithm". I remember implementing the first two in TachyonOS's graphics API, so there should be code for it there.

How much of paint to intend to implement? What is your program going to do?
