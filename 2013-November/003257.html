G'day all,
There's a new update for the Mouse Library that adds a new operating mode.

Changes:
  * Fixed a bug causing mouse movement to glitch on some systems (such as
QEMU).
  * Created library documentation
  * Added event mode, a method of creating event driven programs.

The new event mode allows a program to assign handlers for mouse and
keyboard event and run in an event-driven mode rather than procedural. The
events cover mouse movement, mouse button presses/releases and keyboard
keys. There's a demonstration program, 'mouse3.asm' that shows this new
mode.

Tested on:
VirtualBox --- successful
QEMU --- successful
Bochs --- failed, locks up (possibly configuration?)
Real Hardware (New) --- successful
Real Hardware (Old) --- successful, as long as mouse is native PS/2 or PS/2
emulation is supported.
No laptop tests but I suspect it may not work on non-mouse pointing devices
such as trackballs or trackpads.
