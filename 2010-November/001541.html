Hi all,

Royce, your pgm to shut down the computer should make a few checks after calling
INT 15h with AX=5300h and before carrying on with subsequent calls:

It should check for BX=504Dh (the characters "PM") and that the carry flag is 
not set.

As Sasano pointed out, Advanced Power Management has always been a bit of a 
"dog's breakfast" with many a BIOS
and the BIOS on many new Intel machines don't fully support it and return a 
carry not set ( no error) but don't return BX=504Dh
so your code will not work, making it very machine/BIOS specific.

APM has pretty much been superseded by ACPI, which is a whole different kettle 
of fish.
(see the specs at http://www.acpi.info/DOWNLOADS/ACPIspec40a.pdf , it's a head 
splitting 730 page document)
ACPI uses tables with 32 bit offsets to functions in (very) high memory usualy 
above 4MB so you will have a very hard time accessing them
in a 16 bit P.M. OS like MikeOS.

The INT 20h as Bazza pointed out is not required as it's a DOS interrupt.

In the "old" days you could use the following to turn off some machines:

mov ah, 42h
mov al, 01
int 15h

Paulo.
