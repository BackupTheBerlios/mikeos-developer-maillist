hi,

> My point is that INT 20 is not a BIOS interrupt, it is an MS-DOS interrupt.

How about to replace int20h call (finish program of MS-DOS) to busy loop
or anything else?

For example:

	    mov ax,5307h
	    mov bx,1
	    mov cx,3
	    int 15h
	        int 20h <--

will be

	    mov ax,5307h
	    mov bx,1
	    mov cx,3
	    int 15h
	halt_loop:
	    cli			; disable interrupt
	    hlt
	    jmp halt_loop	; never return!

The aim is giving a chance for safty shutdown, and I think Bazza is 
worrying if BIOS fails to shutdown and code goes to int20h call.

I heard that APM BIOS is sometimes buggy...
