> I know from the OSDev forums that writing USB stacks is a LOT of VERY
> complicated work.

Without a doubt but keep in mind that there is a very big difference between 
writting a USB stack
for say file access and something simpler like keyboard access.

To give you an idea, say you wanted to query the type of device connected to a 
specific USB port.
First get the Base Address Register of the USB controller from PCI ( USB always 
"hangs" off the PCI bus.)
Now that we have the B.A.R.query the USB controller as to which ports are 
currently active (something connected).
Once we know which port/s,  we query the buffer of that port and we get 2 bytes 
which tells us if it's a low speed or full speed device.
Not that difficult.

To read a keyboard (which are always low speed devices) we simply read the 
buffer located in memory (which we also can get the offset from PCI).

Now to make a driver for say accessing a file system on a flash drive is of 
course much, much more complicated.
