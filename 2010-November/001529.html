Well, yeah, that's in the code.  When there is a 'autorun.bas' or
'autorun.bin' it runs, then goes to the app menu.  I've been
constantly looking over the code of MikeOS and Casnix OS.  All you
have to do is press [ESC] to return to the initial menu for CLI.  And
I wrote the autorun.bas on Casnix, so there might be a simple
portability issue.

However, if you're rebuilding MikeOS with the autorun.bas, you might
have to edit build script.

>>
>> Hello, I've created a log in program for my OS based off MikeOS,
>> Casnix OS (http://casnix.sourceforge.net), it's a basic program in
>> 'autorun.bas'.  I thought some people using/developing MikeOS might
>> like it, too.  Here's the source:
>>
>>   rem *** AUTORUN.BAS ***
>>
>>   gosub begin
>>
>>   gosub login
>>
>>   begin:
>>        print "Startup message"
>>   return
>>
>>   login:
>>        goto _msg
>>        goto _prompt
>>
>>   _msg:
>>        print "This PC is protected".
>>
>>   _prompt:
>>        IF C = 1 then CLS
>>        print "Username: ";
>>        input $1
>>        if $1 = "root" then goto .pswd
>>        E = 1
>>        goto .error
>>
>>   .pswd:
>>        print "Password: ";
>>        input $2
>>        if $2 = "root" then goto _start
>>        E = 2
>>        goto .error
>>
>>   .error:
>>        IF E = 1 then alert "Incorrect username!" goto @re
>>        IF E = 2 then alert "Incorrect password!" goto @re
>>
>>   @re:
>>        C = 1
>>        goto _prompt
>>
>>   _start:
>>        end
>>
>>   return
>>
