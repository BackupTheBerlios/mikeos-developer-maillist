<paulovalongo at rocketmail.com> wrote:
>
> Is UEFI that much of a problem?
> I know it would require a total rewrite but I reckon it's do-able.
> Perhaps in the future MikeOS will go this way once traditional BIOS disappears?

The problem is, UEFI doesn't provide drivers for common hardware (or
at least, doesn't guarantee that drivers are available). With the
BIOS, we're guaranteed that we can do disk operations, draw characters
on the screen, get keyboard input etc. easily via BIOS routines. Then
there's the BIOS handling USB keyboards for us too.

Take those away, and we'd have to start writing drivers for all of
these things. Ugly, messy, fiddly PC drivers. Maybe a whole USB stack
to just get keyboards on modern machines working. The code size would
double or triple with really complicated stuff.

Right now, I think the key selling point of MikeOS is that you can
really study how a simple OS works without having to get too bogged
down in the hardware. MikeOS puts OS stuff at the core -- system
calls, loading and executing programs etc. Throw a huge amount of
complicated driver code all over it, and suddenly it loses its
attractiveness.

I don't know a huge amount about EFI, so perhaps things will change,
but it doesn't look promising for this type of OS project :-/
