On Mon, Nov 29, 2010 at 9:08 PM, Paulo Valongo
<paulovalongo at rocketmail.com> wrote:
>
> I was under the impression that UEFI was a complete replacement for BIOS
> in that the machine first boots up to it then onto an OS if it's installed.
> Now in the case of someone wanting to install an OS for the first time,
> keypresses are an integral part of that procedure and considering that
> 99% of new motherboards don't have PS2 ports, UEFI must surely provide some
> sort of generic driver for HID devices (i.e keyboards and rodents).

To what does it need to provide them? Any OS going into protected mode
has its own drivers. The Linux kernel doesn't use any BIOS calls for
keyboard, screen, disk etc. It does it all itself, and I'd wager
99.99% the same is true for the WinNT kernel too (unless it's doing
something VERY crazy with vm86). These OSes don't need the BIOS or EFI
to provide anything for them.

I'm sure EFI systems will have settings screens, like old-school
BIOSes, but I see no reason why they'd need to expose
keyboard/screen/disk facilities to operating systems. The serious OSes
wouldn't use it, and the only projects that need such help are
relatively tiny ones like ours.
