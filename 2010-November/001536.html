Well, when I started designing and writing my own C kernel a while
back, I wrote the kernel in C, made a kern_start program, and linked
the two in the kernel.bin file.

kern_start.asm

[BITS 16]
[global]
[extern _k_main]

start:
     call _k_main  ;start kernel
     cli   ; clear all interupts
     hlt   ; halt process (I think?)

The complete tutorial is here:
http://www.osdever.net/tutorials/view/writing-a-simple-c-kernel

> That's a very, very broad question. So much depends on:
>
> 1) If you're booting from a floppy or hard drive
> 2) The filesystem format of the media in question
> 3) Whether you want to be in real mode or protected mode
>
> For a 16-bit, real-mode OS running from a FAT12 floppy, a great
> example is the MikeOS bootloader code (bootload.asm). Demonstrates how
> to parse the FAT tables on the disk to find a kernel filename, then
> load the sectors to the right point in RAM and jump to that point to
> begin execution.
