On 10/1/08, Troy Martin <blasterman95 at gmail.com> wrote:
>
>  And with the new os_input_string, another great addition that helps a lot
>  with this! How does the multi-line enter and backspace work exactly?

Beforehand, os_input_string was limited to the current line of the
display -- ie 80 chars from the start. If you entered more, it would
wrap round to the next line, but you couldn't backspace round to the
previous line. If you run the CLI in the current release, and enter
enough chars to go onto the next line, then try to backspace back onto
the previous line, you'll see what I mean.

I've updated this now so that you can input and edit lines up to 255
chars. It'd be worth using os_input_string in your editor to keep the
codebase down.

As for Google Talk, the web-based Gmail interface includes the ability
to chat, and I think that can handle more than a two-way conversation.
Not tried it though!
