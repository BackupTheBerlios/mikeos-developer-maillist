Hokay, this is the memory map I'm going to use (apologies again for
the mixture of hex and decimal). The disk buffer has to be in the
first 64K so that it's easy to pass its location to the BIOS when
using disk read/write interrupts -- no need to fiddle with the segment
registers, which should always be 0 (except for CS).

I'm going to work on the bootloader and kernel tonight to fit this
scheme, then start implementing int 47h.


0x0-0x3FF (0-1023)		Interrupt vectors
0x400-0x5FF (1024-1535)		BIOS/system RAM

0x600-0x9FFFF (1536-655359)	MikeOS, of which:

	1536 - 9727		8K stack
	9728 - 17919		8K disk buffer

	17920 - 83455		64K external program segment

	83456 - 148991		64K kernel code segment

	148991 - 655359		494.5K general RAM for external programs

0xA0000-0xBFFFF (655360+)	Video RAM
