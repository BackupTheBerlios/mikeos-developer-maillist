I think passing parameters via stack is 
easy for making glue logic (for example libimikeos).

For example, in C language, calling

	short function(int parameter1, int parameter2)

is translated to this.

	push parameter2
	push parameter1
	call _function
	add sp, 4

So, glue makers do not have to think what register is
used for API's argument. But MikeOS is assembler-friendly,
please consider assembler-programmer first.
