Now I am testing os_mouse_xxxxx function of libmikeos.
I will release it when the test (API glue) with MikeOS 2.0.0-rc has passed.

http://www.bi.a.u-tokyo.ac.jp/~uaa/gomitext/2008/20081013/libmikeos-0.1.1-rc.tar.gz


By the way, the comment of os_mouse_wait_for_l_click says

> ; OUT: AX - x postion of click
> ;      BX - y position of click

but the test result shows AX -> Y positon, BX -> X position.

The mouse BIOS always use CX as X and DX as Y, so the code of
os_mouse_wait_for_l_click

>	mov ax, dx
>	mov bx, cx
>	pop dx
>	pop cx
>	ret

might be

>	mov bx, dx
>	mov ax, cx
>	pop dx
>	pop cx
>	ret

I think.

(I use libmikeos-0.1.1-rc/sample/test0d/test0d.bin for test this API.)
