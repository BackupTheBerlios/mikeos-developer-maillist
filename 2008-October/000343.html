Grr! I just managed to get it to work, and noticed that there was no bug -
it was just my old laptop being slow. D'oh!

I used si and di to define start and end points. Each time the string didn't
equal '.BIN', si was incremented (di was stored in bl and si in al). The
idea is that once it gets to the stage of being 4 characters away from the
end, if it was finally equal to '.BIN', then it breaks out. If it doesn't,
it keeps incrementing until si and di are the same and an error message is
sent out. However, as MikeOS does a 'Program Terminated' message anyway, my
code suddenly becomes very redundant!

However, I still feel it's a valuable learning experience - I now understand
how to do IF()s and loops in ASM, so I'll be far more confident using it
inline on C projects (like ApolloOS) or making actual ASM apps.
