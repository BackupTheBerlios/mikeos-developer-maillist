Yes.

Many compiler is based on "memory model" and
bcc is a tiny/small model compiler, at least DS has to be same as SS.

It is very hard task for me to explain "memory model",
please refer this web page:- 

http://www.digitalmars.com/ctg/ctgMemoryModel.html


Currently libmikeos uses tiny model with a trick to switch DS/SS
(by the value of SP). But the Mike's new memory map is difficult to
apply this trick.


The new memory map is well-planned, so I am thinking simply using
"user code segment" with tiny model.

This means CS=DS=SS points "user code segment" while
running C-based program, can I do that?

(OS API is invoked by INT instruction, so CS points kernel segment
while OS-routine is running. But where DS/SS points?)
