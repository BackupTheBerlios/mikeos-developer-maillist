> Hi all,
>
> This is the final pre-2.0 release to include new features; from here
> onwards it's just bugfixes. Takayoshi: I have included two new system
> calls that you may want to include in the C API, but there won't be
> any more between now and 2.0, so you can stabilise your work. I'm
> really looking forward to having it as a feature for MikeOS 2.0 --
> thanks!
>
> Main things in this release: I've written a small file manager for
> deleting and renaming files. Copying would need more RAM, something
> which needs to be addressed for 3.0 (unreal mode might be the way to
> go). Also, this version includes a generic os_list_dialog system call
> which looks much like the file selector, but can be used for other
> purposes.
>
> Full changelog at the end of this mail -- you can grab the code from:
>
> http://prdownload.berlios.de/mikeos/mikeos-1.9.4.tar.gz
>
> Now it's time for bugfixes, comments and more bugfixes! If anyone
> wants to poke around the code and clean up any glitches or add safety
> checks, that'd be grand.
>
> M
>
>
> # New file manager: programs/fileman.asm -- lets you delete files,
> rename files and get information on the floppy disk
>
> # New system call: os_list_dialog -- takes a zero-terminated string
> containing comma-separated options that are displayed to the user in a
> list box; returns number of chosen option (starting from 1)
>
> # os_file_selector is now a wrapper around os_list_dialog
>
> # New system call: os_filename_convert -- takes a normal filename such
> as 'FOO.BIN' and converts it into the FAT12 equivalent of 'FOO
> BIN' (first part padded out with spaces to 8 chars, no full-stop
> separator)
>
> # os_load_file now returns byte count in BX rather than sectors
>
> # Cursor is always restored when leaving os_file_selector
>
> # os_dump_registers now shows SI and DI along with AX, BX, CX, DX
>
> # Removed programs/filetest.asm
>
