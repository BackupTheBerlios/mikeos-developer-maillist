>
> I'm still using some public domain code to get me started on a FAT driver
> though as I haven't a clue where to start

You need a good spec doc, plenty of time to code, and a plan. I knew
very little about FAT12 a month-ish ago -- the MikeOS file read code
was from someone else. I hoped someone would step up to implement file
writing, but nothing happened (apart from a never-to-be-completed
TroyProject(tm) :-) ) so I just had to knuckle down and write the
code. A lot to learn!

Here's the spec file I used:

http://www.it.lth.se/ssoa/Project1/FAT12Description.pdf

That's all you need. Start by understanding the concepts of FAT12
disks -- the FAT table, the root directory, and the data sectors. Make
sure you can link together an entry in the root directory with the
starting cluster in the FAT section, and then the data parts of the
disk. Forget writing for now -- it's a lot harder. Once you've got
file reading implemented, you'll be confident enough with FAT12 to
tackle writing.

My FAT12 write code will have a lot of cleanups, comments and
documentation before MikeOS 2.0, so hopefully that will be a good
starting point, along with that excellent PDF above!
