On Tue, Oct 21, 2008 at 12:53 PM, SASANO Takayoshi <uaa at mx5.nisiq.net> wrote:
>
> Well, will you have a plan to change API calling convention?

Nope, I'd like to keep it pretty much exactly the same: external
programs call vectors at the start of the kernel. However, because the
programs and kernel will be in separate segments, we'll have to use
far calls (I think).

So in mikedev.inc it'd be:

os_string_uppercase     equ     0x2000:0x8030  ; AX = zero-terminated string

And for the vectors in os_main we'd have:

        jmp os_string_uppercase         ; 0x2000:0x8030

System calls will have to finish with 'retf' I believe. Again, I've
only briefly looked into unreal mode so I'm not completely sure that
this is all correct, but I guess we'll find out soon!
