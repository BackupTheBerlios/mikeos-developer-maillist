Hi all,

With 2.0 safely out the door, we can start thinking about plans for
the next few releases. Currently we have in the pipeline:

2.1:
 - Debugger (Tomasz)
 - Text editor (me)

There'll a few new system calls and the usual bugfixes too. Does
anyone want to add anything else to that list?

Meanwhile, for 3.0 I have some ideas:

1) Unreal mode. MikeOS and its programs are getting pretty cramped in
that 64K of space. By using unreal mode, we can use full 32-bit
offsets for data, although CS:IP is stuck with 64K segments as far as
I can see.

However, it means we could have (A) a 64K kernel segment; (B) a 64K
external program segment; and (C) 512K of RAM for the external
programs to use. This would involve going through the MikeOS source
code and changing all offsets to use 32-bit registers (eg ESI instead
of SI) but I don't think it'd be too hard.

2) Reorganising the codebase. Right now we have source/ with a bunch
of files that are pulled in by os_main.asm. I think it would be
cleaner to have:

source/kernel.asm
source/components/
source/components/mouse.asm
source/components/floppy_disk.asm
source/components/screen.asm

and so forth.

3) New documentation. The Handbook is pretty huge and covers a lot, so
maybe we can split it up like this:

handbook-user.html -- For people running MikeOS (on real PCs and in an emulator)
handbook-appdev.html -- Explains how to write MikeOS programs, with a
detailed list of the available system calls
handbook-sysdev.html -- Documentation on building MikeOS, editing the
code, adding system calls, making patches etc.

Any comments welcome!

Cheers,
Mike
