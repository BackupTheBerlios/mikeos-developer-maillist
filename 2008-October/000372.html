Righto, we'll see how it goes then. There might be a problem with my
unreal mode plan -- even if we can use 32-bit offsets, can the BIOS
handle that? For instance, to write sectors we have int13h / ah=03:

http://www.ctyme.com/intr/rb-0608.htm

The data location is passed in BX. To make use of that new 512K of
free RAM, we'd want to be able to pass the data location in EBX. If
the BIOS still only uses 16-bit offsets, it's all pointless as I see
it! Doh.

I'll do some experimenting this evening. If it all fails, maybe we'll
have to just accept that segments are here to stay, and try to make
those as simple as possible (ie a kernel segment, a program segment,
and a RAM segment).
