Hi Takayoshi,

I think I'm starting to understand. (I never did any DOS programming,
so the concept of memory models is a bit unusual to me! Hence why I
was so adamant to put everything in one segment in the original MikeOS
:-) )

> The new memory map is well-planned, so I am thinking simply using
> "user code segment" with tiny model.
>
> This means CS=DS=SS points "user code segment" while
> running C-based program, can I do that?

Technically, in the startup code for C programs you could point DS and
SS at CS and thereby have them all pointing to the same segment. So
that might be possible, providing you reset DS back to 0 when the
program exits.

How about using a different compiler? According to some mailing list
posts, GCC can generate code for use in unreal mode, providing you
pass all the usual --nostdlib --no-nothing-else-ever etc. flags.
