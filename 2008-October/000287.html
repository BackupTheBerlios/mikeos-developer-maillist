PROGRESS! I'm still working on the CLI and the floppy driver, but we have an
ApolloOS that can handle memory, output to the screen and handle interrupts.
It also runs a timer (but stops displaying output after 10 seconds). This
could mean I could delay boot ups for splash screens/GUIs or even to help
tell the user how long their session has lasted.

It's taken a while, but we're not far off a 0.1 release. I'm releasing the
current version as ApolloOS 0.0.5 on the basis that I haven't added the
stuff that will distinguish it as an OS (i.e. CLI, floppy driver, and in
later versions a GUI and a mouse driver). It feels a lot like a slightly
modified bkerndev at the moment despite being built on public domain code. I
will be porting MikeOS modules/apps/drivers shortly, so there will be
justification in my claiming ApolloOS to be 'similar'!

You can grab it via this link, though it will be on SF.NET as soon as I can
get it on there:
http://apolloopsys.googlepages.com/apolloOS_0.0.5.tar.gz
