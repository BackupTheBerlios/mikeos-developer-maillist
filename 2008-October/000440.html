Ok - so a program can be 64KB in size and use 495K for doing interesting
stuff (that's a lot more flexible than the old model). To get it working in
C you just need to set the memory addresses as constants and work from there
(though I'm braving ASM at the moment to see how it goes!)

I suppose you could give the 'illusion' of multitasking (i.e. running one
binary, which loads two other binaries to the 495K segment (taking up 128K
respectively). You could then then run the other binaries as requested, and
the initial binary would use more on top of the 128K to store information on
the app's state, etc. as you switched between them), but that would be quite
complicated! :-)
