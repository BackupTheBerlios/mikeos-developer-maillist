> Interesting! What do you see it being used for, though, without
> multitasking?

In Handbook is written "
(We also have a blank 8K of space at the top of
our kernel segment - it's an empty area that programs can use as a
buffer. Care should be exercised with its use since other subroutines
may be using it, too"

I meant to write a mechanism to unify memory managment in mikeos.
Then we wouldn't have to worry that a procedure could overwrite our
data.
It doesnt have to be complex stuff just a simple system to manage
memory.
Then the user doesn't have to know that free memory starts at
2000:E000.

Just make it possible to "ask" the system about a certain amount of
memory for example:
   
    mov ax, 15       ; number of bytes to alloc
    call _mm_alloc   ; alloc it
    test ax, ax
    jz .no_memory
        ; here in ES:DI we have addres of memory we can use.
