Which bit don't you get? The table of "jmp" instructions at the start
of the kernel stays in the same place. For instance, if you want to
use os_print_string in your application, you don't know where the code
for that routine is in the kernel. It could be at byte 13257 in MikeOS
4.2, but if we add some new code in MikeOS 4.3, it could move to
14283. The location of the routine can shift around.

However, by using a list of "jmp" instructions at the start of the
kernel, they always stay in the same place and never move, so external
programs can "call" them.
