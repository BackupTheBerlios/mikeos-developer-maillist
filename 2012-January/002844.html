1) To run DOS COM programs you'd need to add a hook to int 21h and
then recreate all the DOS calls required to run your programs. It's a
lot of work. We had this in MikeOS 2.0, but it was complicated,
half-implemented and didn't make sense in this kind of OS.

2) Segments are utterly horrible to deal with in a tutorial, learning
OS like MikeOS. It's far easier to explain how to modify the OS and
write programs when everything is sitting together in the same memory
segment, I believe.
