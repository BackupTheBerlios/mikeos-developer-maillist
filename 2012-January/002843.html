I was trying to design a COM program loader. The way it was supposed to work is that MikeOS would open the 32k program LOADCOM.BIN as usual, display a file selection menu, and load the MS-DOS COM file into the next 64k segment. (3000h) After much difficulty and finally realizing that os_load_file doesn’t support an option to load the file into another segment and discovering no other possible workaround, I gave up.

The 32k limit for programs is sufficient for most of your typical hobby OS programs, but I don’t understand why MikeOS wasn’t designed to allow programs a full 64k block in the first place. Program development would’ve certainly been easier had it been set up that way.
