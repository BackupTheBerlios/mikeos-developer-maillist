>There's lots of other "state" associated with a running BASIC program
>though -- like the data stored in for_code_points, gosub_points etc.

I've already figured all that out.

The first part of the assembly code in Yotta controls the BASIC code.
It preforms tasks such as decrypting, starting and detecting crashes.
When a BASIC program is to be run, the main BASIC code will be
ended and control will be passed to this 'visor' to run the file being edited.

Afterwards, the main BASIC code will be restart will a special
starting condition, in this it will restore all the variables and continue.
Since the code will be effectively starting again there will be no need
to worry about for_code_points or anything like that.
