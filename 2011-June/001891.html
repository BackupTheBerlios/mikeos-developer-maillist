Great, i'm glad it's one step closer to running on the 8086 :)

To explain my motivation a bit: I have very fond memories of my first PC, a
Schneider Euro PC II (8086-based with 640kb RAM), so I have been
entertaining the idea of trying to run Mike OS on it as a fun
side-project. While all the other OSes target modern machines (as they
should), to me Mike OS seems to be the perfect hobbyist OS to try and
install on low end machines, maybe as low end as the 8086 some day, given
some modifications.

I should have that 8086 PC along with a 486 at my parents', next time i get
there i'll see if those machines are still in working condition.

Mike OS basic is also fun and takes me back to the days of my ZX Spectrum
(the computer i used before my 8086 in fact). The syntax of MikeOS Basic and
Sinclair basic seem to be similar enough. I like some of the decisions you
made, "Call <address>" is definitely more clean than Sinclair's "Randomize
Usr <address>" :)  The only thing that confused me at first was the POKE
command, in Sinclair Basic the address and byte order is reversed ("POKE
<address>, <byte>")

Anyway enough rambling for now. Great project you have here Mike! I'm
surprised it is not more popular among retro hobbyists such as myself...

Bazza, thanks for all the information you provided, very useful indeed!

Best,

Mario
