I actually like this idea :) Booting right into BASIC and having a BASIC
console like on the Spectrum seems a neat idea.

To add to the brainstorm a bit, at some point it might be interesting to
support loading BASIC bytecode and binary code (similar to Sinclair's LOAD
"" and LOAD "" CODE). As far as I can tell, you already have the later
functionality, but the former might be useful for loading programs from a
BASIC console, or as part of a larger program. I haven't looked at the
source for your interpreter yet, but i'm guessing substantial changes might
be needed.

Well, just throwing the idea out there.

Also, a small but useful addition to MikeOS Basic might be a BIN function
similar to the one on the Sinclair Spectrum. Something like:

print bin 1111   ---> 15

In fact, once I get to my vintage hardware I might take a crack at
implementing this (unless someone wants to do it first, of course). It would
be a good chance to get at least a bit familiar with the codebase.

I see interesting possibilities ... :)

Mario




> One idea I had, a while back, was to make a spin-off version of MikeOS
> that booted straight into BASIC. Users could write it to a USB key,
> boot it up and explore, like on the old 8-bit machines. People could
> give it to their kids, so the young 'uns could get a sense of
> controlling the machine, poking and peeking bytes, like how we learnt.
>
> I think there are a few things missing from MikeOS BASIC to make that
> doable though: READ + DATA, proper array declaration and handling, and
> arbitrary variable names. If I ever get round to implementing them, I
> might start BasicOS as a side project and see how it goes!
