Hi folks,

It's here: http://prdownload.berlios.de/mikeos/mikeos-4.3b1.zip

Aside from Mike Gonta's updates for USB drive floppy emulation, there
are various new BASIC goodies here. Please take the time to read
through this carefully before embarking on a new BASIC project!


1) GRAPHICS IN BASIC

Yes, you can now do some simple graphics stuff in BASIC. First switch
to graphics mode using "MODE 19" (there are other modes listed at
http://www.ctyme.com/intr/rb-0069.htm). This puts you in 320x200x256
mode. You can then plot pixels with "PLOT" followed by the X
co-ordinate, Y co-ordinate and colour (can be variables). You can also
draw boxes using the BOX command (with start X, start Y, end X, end Y,
colour). Here's an example:

--- CODE ---
mode 19

y = 150

for x = 1 to 70
  box x x y y x
  y = y - 1
next x

waitkey x

mode 3
end
--- CODE ---

Note that you should "MODE 3" at the end to return to normal text
mode. This demo program results in this:
http://mikeos.berlios.de/basic_gfx.png


2) COLOURED TEXT WITH INK

Using the "INK" command followed by a number or variable, you can
determine the colour of text strings and text variables in the PRINT
command.

--- CODE ---
ink 4
print "This is some red text!"
--- CODE ---


3) DATA READING -- NO MORE POKES!

There's a new "READ" command that reads bytes from a label, with an
offset, into a variable. For example:

--- CODE ---
y = 1

for x = 50000 to 50012
  read mydata y a
  poke a x
  y = y + 1
next x

call 50000

waitkey x
end

mydata:
190 87 195 232 173 60 195 89 111 33 13 10 0
--- CODE ---

Here, "mydata" is a label with some numbers after it (decimal only at
the moment). Those numbers represent a tiny program ORGed to 50000
that prints "Yo!" and rets out. For the "READ" command, we specify
first the label, then the offset from the label (it starts from 1 for
the first number), and then the variable into which we want to read
it.


4) AND ADDED TO IF

Pretty simple this one -- you can now do "if x = 1 and y = 2 then blah" etc.


OK, that's everything for now. Lots of stuff to play around with! Lots
of limitations of course -- please don't post asking for lines,
circles, filled shapes etc. I'll do them when I have time. If you want
something, write the code!
