Not move.
It normally seems to go to the next line causing:
MOVE 79 24
J = 219
PRINT CHR J ;
To scroll to the next line even though it is follow by ";". This makes it
impossible to draw a full screen border. Notice that Muncher or MikeTron
cannot do a full border because of this.

MB++ does this in the BORDER function because it uses an assembly code for
printing text. This was so it could print in colour before it was available
through BASIC commands. It had the side affect of not moving the cursor.
That allow us to print in any position without worrying about scrolling.

However I'm removing this from MB++ as part of a major re-write and using
"INK" with standard text printing commands, which make the code faster and
easier to read. Now I've realized that it will be impossible to draw the
border without scrolling.

I'm asking that printing to column 79 (last column) with the ";" on the end
should not cause it to do to the next line especially not on line 24 (last
line), rather stay at that position. This would very much benefit applications.
