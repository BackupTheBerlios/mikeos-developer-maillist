>How is using L for len any different than R or >S for save and load?
It removes another variable I could use. Don't think I like load/save using
R and S. I have to assign their values to throw away variables temporarily
meaning I run out of then and have to resort to peek and poke.
Many of my programs use the 'L' variable and couldn't take advantage of LEN.

I'm sure it's more convenient to use 'L' but if every function used a
variable there would be none left. I think I should be about to specify a
variable to put it to. Like "LEN $1 J" to put the length of $1 into J.
