And more fun for today! I was just looking at Joshua's page and saw
that he's done some great work on a wiki here:

http://mikebasicplusplus.wikispot.org/Getting_Started

And there's a reference for functions at:

http://mikebasicplusplus.wikispot.org/Front_Page

I highly recommend MikeBASIC developers check it out and use it in
their apps! It's coming along really well. Which prompts me to think
about including it in 4.3, so this is some stuff for Joshua to think
about and discuss on the list...

Joshua, how doable is this: we put MBPP.BAS containing your
MikeBASIC++ routines on the disk in 4.3. If the user runs it directly,
it just prints "This is a library - not for direct use" or whatever.
However, for any program that wants to use it, then can do this:

--- CODE ---
x = RAMSTART
load "mbpp.bas" x
x = x + s

rem Now X = start of free RAM, after the end of MBPP
--- CODE ---

So ultimately, MBPP.BAS would be a like a library that BASIC programs
can load onto the end of themselves, and then call routines inside it.
This might require some changes to the workings of the BASIC
interpreter, but could be worth it. What do you think?
