Mike,

>That's really cool, but you can do something similar in QEMU --

I suspected that there might be a way to do it in QEMU, however
I didn't bother for several reasons:

- I've never got round to loading and learning QEMU
- It will take longer to learn QEMU then to fire up the soldering iron
  and make up a null modem cable.
- QEMU is not always available, especially with real hardware.

It would be nice to include a debug_message type call in MikeOS
that can be used in situations where one needs to see how far a pgm
gets before crashing.
For example it could send 2 bytes to the serial port at various locations
of a pgm.

e.g.
; first initialise the serial port (only required once before first call)

mov ax, 0
call os_serial_port_enable

; now we put one ascii char in al and another in ah
; this will allow for 100 different "codes" indicating
; where about in the code the pgm is.

mov al, '30h'
mov ah, '31h'
call os_send_debug_msg

this would send the ascii chars 01 to the serial port.

Windows has a OutputDebugString in the kernel and it's very handy.
Obviously in windows, the debug message would be captured by
another app listening out for them.
In MikeOS we don't have this luxury so we could send it to the serial port.

Serial null modem cables are stocked by the better computer stores so
those that either can't or don't want to make one could buy one.

Paulo.
