Mike,

So if the iso works, then the ima is also
correct which means the bootloader and
BPB are also ok.

I don't know why your images are different.

I know it's not a priority to convert to FAT16
just yet, but when/if you do, then seriously
consider standardizing on an image size
of 512MB.

It's not too big and when zipped it's down to
about 500Kb making it easy for downloads.

So what's the way forward?

There are FAT16 write and read routines to write
and you might even have to make some changes
to the memory layout of MikeOS as the current
buffer is way too small for FAT16 stuff.

The recommended files buffer for FAT16 seems to
be around 128KB and to make things easier all
files must be in the root of the drive, no directories.

A lot of work still ahead.

Paulo.
