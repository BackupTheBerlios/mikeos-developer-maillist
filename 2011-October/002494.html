Yes, it would require a bit of work but it's a
good idea! At the moment that would print 20 lines
not 20 bytes though, since we only handle text files.
Unfortunately we cannot overwrite files. This is the
reason behind limitations on this in ASCII Artist
and Memory Manipulator. The programs will present an
error message, like "Cannot create file.".
If you have a file called foo.bar and you do
"save foo.bar 12345" then it will set R to 1 and
not do the save. This is the same as an illegal filename
so my programs will treat it in the same way.

If you were to fix this problem, so we could overwrite
files then I would create this, but right now I can't.
Maybe we could have an command to verify if a file exists.
