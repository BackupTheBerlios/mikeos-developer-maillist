On Sat, Oct 1, 2011 at 1:32 AM, Troy Martin <blacklight at cordilon.net> wrote:
>
> Speaking of which, would it be too much trouble to have BASIC's END
> instruction place the contents of the variable A into AX?

That gets very messy, very quickly. Having stuff loitering around in
registers between programs is not a good idea. Programs should launch
with a clean slate, with the exception of command line parameters that
are passed explicitly.

If you leave other stuff in registers, you have to make all sorts of
guesses and assumptions. Program B launches, and finds AX isn't zero
-- does that mean anything? Was a previous program trying to pass it
something? Did the previous program not actually intend that, and
authors should put "A = 0" before "END" if they don't want a program
to accidentally pick up stuff?

So it's not a good idea. There may be a case for an exit code some day
(success/fail) but not like this.
