Hello Rob,

Don't quote me on this as I haven't looked at the code and could be completely wrong but...

> When I try the programme Mouse API Test, moving my wireless mouse
> doesn't work, and moving my touchpad seems to crash the system (cannot
> exit the application, cannot use Control-Alt-Delete to restart).

If I'm not mistaken it will because of the protocol being used and legacy support. I think these are PS/2 mice drivers and you're trying to use this through USB or some other method of attaching your wireless mouse.

The reason for it crashing is probably the fact that it doesn't handle the mouse not being found very well and proceeds to keep searching for it until it works which in this case it wouldn't.

Due to you using a touchpad, I'm assuming you are running a laptop where it's going to be reasonably likely that the hobby OS doesn't have drivers for your hardware in any case. (Your main stream OS, be it Windows, Mac, Linux or whatever does a lot in the background to help get you going in terms of drivers).

Your best bet will be to look through the BIOS and see if there is a switch for legacy mode to be turned on. The only other way of getting around this if it is the problem is using a Virtual Machine where there will almost certainly be a way you can set it up to emulate your mouse as a PS/2 mouse.

Hope that helps, Dan
