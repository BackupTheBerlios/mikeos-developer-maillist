Hmmmmmmmmmmmmmmmmm.........................

It is very difficult to distinguish the pointer is under DS or SS.

        xxxx:0000 +----------------------+
                  | Kernel code          |
        xxxx:8000 +----------------------+----------------------+
                  | User (program) code  | Stack area   (SS)    |
        xxxx:F000 + - - - - - - - - - - -+----------------------+
        xxxx:FFFF +----------------------+

As a member in "embedded" computer world, I will fit in this style...
But, it will be unhappy result to everyone, especially for PC user.

        xxxx:0000 +----------------------+
                  | Kernel code          |
        xxxx:8000 +----------------------+
                  | User (program) code  |
        xxxx:C000 +----------------------+----------------------+
                  |                      | Stack area   (SS)    |
        xxxx:F000 +----------------------+----------------------+

                                MikeOS-2.0  MikeOS-3.0
        Maximum User code size: 32k         16k
        Stack size:             28k         12k

Of course the ratio of user-code/stack can be changed,
but this size is easy to check (with bit-check instruction).


It is the best way to provide 64kbyte segment to application, I think.
