Hi all,

This continues the code cleanup work; most notably, some of the FAT12
calls that were accessible to user programs are now made internal to
source/features/disk.asm to simplify the API. Now user programs can
load, rename, delete etc. files, but not mess around with the root
directory or FAT.

Download: http://prdownload.berlios.de/mikeos/mikeos-2.9.5.zip

There are a few things to be done before 3.0 -- the biggest of which
is the documentation overhaul, which I'm going to work on in the
upcoming week. I'd like to provide full documentation for the API, so
Paul, if you're still up for it, fancy making a HTML page of one of
the files in source/features/? I'm thinking along the lines of:

[h3]os_foo_bar[/h3]

[p][strong]Overclocks the CPU 800x[/strong][/p]

[p][ul]
[li]IN: AX = number of kittens[/li]
[li]OUT: DX = kitten cuteness ratio[/li]
[/ul][/p]

[p]Example:[/p]

[pre]code here[/pre]

[hr /]

Something like that kind of template for all the calls. Or indeed, if
anyone else who knows the API and code wants to give it a shot, great!

More things do to:

- Current FAT12 code only writes to the first FAT, and not the backup one

- In FILEMAN.BIN you can rename a file and give it the same name as an
existing file

- The os_file_create currently doesn't set the real time/date for the
file creation in the root directory entry

- Previous MikeOS releases included a game :-) Not 3.0 though, as
we've removed the DOS compatibility. But it'd still be good to include
a game for 3.0, whether written from scratch or a port of a (free!)
DOS game.

Cheers,
Mike
