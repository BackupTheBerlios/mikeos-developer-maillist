Hi.

finally I have started writing code for TomOS and I have found small
issue with the build-linux.sh file.

When you remove everything from disk_images directory MikeOS won't
compile complaining 'No space left on device'. It is caused by usiing
dd for creating 'mikeos.flp'.

Can be fixed by using for example mkdosfs to create a floppy image:

mkdosfs -C  disk_images/mikeos.flp 1440  || exit

then afterwards use dd to add bootload.bin as usuall:

dd status=noxfer conv=notrunc if=source/bootload/bootload.bin
of=disk_images/mikeos.flp || exit

It isn't a big deal and MikeOS can live with that :)
