>    In the following bootloader code.
> 
>     cli
>     mov ss,ax
>     mov sp,4096
>     sti
> 
>    there is clearing and setting of interrupts to prevent an interrupt from
> taking place between the 2 mov instructions. Now when a machine boots where
> does SS and SP point to? Does the BIOS set some stack space? If it doesn't
> shouldn't it clear the interrupt at the start of the code and set the
> interrupt only after the stack is set.

If  you debug the BIOS this starts at 16 bytes in from the END of the last REAL
MODE segment this is at FFFF:0, (F000:FFF0).

The instruction WILL ALWAYS be JMP F000:E05B... This will probably point to
another jump instruction INSIDE segment/offset F000:0000
Keep going to the start of the BIOS routines and these start with CLI anyhow...

Before ANY BOOT can occur there MUST be 64KB of RAM at segment:offest
0000:0000...

In MikeOSs case CS: and SS: sit at this point.
You can ignore FS: and GS: as in the "real world" these do not exist... ;o)
