Hi Mike,

Actually maths routines add a whole bunch of problems and potential pitfalls and compatibility
issues if a strict format is not followed.

For example, let's take a hypothetical os_call named os_get_sine
Where the input would be an angle and output would be the sine thereof.
Irrespective whether the actual calculations take place in a library or  the FPU,

1) What format would the input be in ? (i.e how would one pass say 12.75) Hundreds in say AX, tens in BX, etc etc
2) What format would the output be in ? Hundreds in AX, tens in BX, etc etc
3) How many decimal points would be sufficient ? (Bazza hinted at 3 -- seems enough)

It would be impractical/off-putting to expect someone to now have to go do a whole bunch more calculations just to convert
the output from floating point to "normal" numbers.

What I'm trying to get at is, perhaps you should draw up a format which would be asm and basic "friendly" and any maths submissions would have to comply with that format.

Paulo.
