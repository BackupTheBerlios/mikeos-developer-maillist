> I would hate MikeOS being abandoned for a fork of it, and would
> feel guilty, as it seems to be only me that uses Mike BASIC in anger.

Worry not -- MikeOS will never be abandoned. I'll always be around to
add bugfixes, updates etc. I'm happy with the code and docs as they
are now, so I have no plans to add anything else myself, and MikeOS
may well be still at 4.1.x in 12 months. But then I never planned to
write a BASIC interpreter, and we don't know if anyone will submit new
features one day :-)

Right now, the MikeOS BASIC interpreter is one component of a very
technical OS development learning package. It'd be great to have more
BASIC programs, but right now the folks who download MikeOS are
looking for asm code and ideas -- they're unlikely to be interested in
anything high level.

If we rip this out and make an OS with the BASIC interpreter
centre-stage, however, it may attract a very different type of user...
