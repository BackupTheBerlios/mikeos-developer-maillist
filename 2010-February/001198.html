Hi All...

> The x87 was only optional up to the 486.
> From there on it's part of the main CPU.
> But of course Intel had to make things complicated and the FPU 
> does not use normal registers like the x86.
> Instead it uses 8 x 80 bit linear "addresses".

> You can still use the FPU on a 16 bit OS.
> The x87 will write a 32 or 64 bit number to memory and you could use
> the opcodes CWD (convert word to double word) and
> CWDE (convert word to extended double word) to simplify things.

All good stuff except...

BEAWRE...

1) We have only (pseudo-)64KB of memory to play with.
2) There weren`t many - if any - 8088/8086 machines with an 8087.
3) If loaded libraries are used then there are only two segments
available - the "user" one at 2000:0 and the boot one at 0000:0
This already makes 128KB of available memory so MikeOS is in
danger of slipping into a serious OS tool instead of a learning tool.
4) If we want as a group the full base memory of 640 (+ 384) KB
then fine by me, but the OS is "sold" on the MikeOS site for its primary
restriction of 64KB, which is what attracted me to it in the first place.
5) As I have said before it already is not true 8088 16 bit code.
6) If 4) is valid then why not generate dedicated MikeOS interrupt vectors
like MS-DOSs INT 21 and on and have much more flexibility such as USB 1.1
support as a minimum...

Just my 5 pennoth...
