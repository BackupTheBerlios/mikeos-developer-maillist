On a more technical note, I have a few question type things:

- What OS calls does BASIC /actually need/ and which can be recoded in
  basic?
- I firmly suggest a move to fasm for a number of reasons (smaller
  binaries, better macros, larger OSDev community ...)
- What, if any, residual kernel functions still need to be there?
- Can we design an interface between BASIC and BIOS so that we can have
  nearly everything be interpreted in BASIC, incl. char print etc
- Should we just have a bootloader set up the interp+some .bas files?
- Should we have (now this may be a tad extreme) a proto-BASIC that
  interprntes the rules of BASIC to allow it to be self modifying? EG,
  redefine addition or something
- Most importantly, what'll it be called? Something along the lines of
  Simplos? Beginos? Basos?

I take it we're going ahead with a full fork here, any takers? Oh, and
count me in.
