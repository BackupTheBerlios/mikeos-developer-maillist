I don't mean that in negative way -- what I mean is, if we want people
to use our BASIC interpreter, we need to put it in a different OS than
an asm-focused teaching tool. If we make a MikeOS fork focused on
BASIC, a ZX Spectrum-like OS, we can do all the stuff we want with the
interpreter, and if that means people run it and send us BASIC
programs they've written, all the better!

If such a thing ever gets off the ground, then yes, it'll have its own
name and site and mailing lists. And if it was reasonably popular
enough, there'd be scope for a hackers list (like this one, for
developing the OS) and a BASIC list (for those writing BASIC
programs).

Right now we have a BASIC that's pretty limited but usable for writing
small games, file tools etc. Few people are going to try it because
it's tucked away in an asm-oriented OS learning project. Put into its
own OS and expanded, it has more chance for people to use it -- we
have a new project to work on, and it'd get better as people submitted
their BASIC programs too :-)
