Hi Paulo,

Interesting ideas, but in the name of simplicity I'd rather not
conflate the two different programming approaches. BASIC should stand
on its own: it should evolve to support the needs of programmers
rather than to awkwardly have bits of assembler shoe-horned in.

After all, there's already a defined API for asm developers so I don't
think BASIC needs to compensate for anything. Simple asm operations
can also be accomplished by, eg:

POKE 250 65500
POKE 195 65501
CALL 65500

to execute CLI then RET.

> reg.AH = 8
> reg.BH = 0
> INT(10h)
> a = (reg.AL)   ; character returned in al
> b = (reg.AH)   ; character's colour returned in ah

The best thing here would be to implement it as a BASIC instruction.
Ultimately, if you really want to do something in BASIC but it's only
possible through falling back to asm, then the BASIC interpreter is
missing something and we should implement it in BASIC.

I'm up for adding more BASIC instructions for MikeOS 4.2 so fire away!
Within reason of course. An XML parser might be beyond my nous :-)
