This suggests we need some kind of READ X facility as in BASICs of old
with accompanying DATA bits. What do you reckon folks?

For instance, imagine DATA here contains some machine code (I've just
made it up), and READ pulls a byte from the DATA section and
increments the DATA section pointer. If it's the end of the data, the
R variable contains 1. So we POKE bytes from the DATA section into RAM
and then call it -- just like old BASIC routines for inputting hex
from magazine listings. I'm sure it'd have lots of uses beyond
prodding machine code into RAM too!


y = 60000

loopy:
  read x
  if r = 1 then goto endloop
  poke x y
  y = y + 1
  goto loopy

endloop:
  call 60000
  end
