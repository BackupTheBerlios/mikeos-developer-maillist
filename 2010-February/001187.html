In the mean time I have been playing around with the X87 math co-processor:

; ========================================================================
; Simple x87 math co-processor test. 
; var_one contains number from which square root will be taken.
; var_two contains answer (32 bit) rounded off to nearest unit.
; compile with A86 and run under DEBUG only.
;
; once run under DEBUG with g=100 the answer var_two is at offset 0113h
;
; The square root of 250 is 15.811388 rounded off it's 16 (10h)
; which is exactly what you should see at offset 0113h i.e. var_two
; The reason I went for rounding off is that it's OK for most simple apps
; and have you tried converting/working with single precision floating
; point numbers?  not pleasant.
; 
; Paulo 09 Feb 2010
; ========================================================================

finit
fild Dword var_one
fsqrt
fistp Dword var_two
fwait

int 3h

var_one DD 250
var_two DD 0

; ========================================================================
