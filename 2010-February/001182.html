> Still leaves SIN, COS and TAN though.

MikeOS is not floating point friendly so you will probably have to
approximate using whole number fractions and/or whole number
degree angles and use a look up table.
Don`t forget divide by zero for TAN.

For my use I have solved - within limits - how to get 2 decimal
places for a digital Voltmeter idea using MikeOS. A little convoluted
but it works and will probably use it in the Arduino project coming soon(ish).
