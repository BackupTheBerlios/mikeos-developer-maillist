> Wow! From a quick look, and perhaps I'm missing something here, the
> kernel still appears to use BIOS interrupts for screen handling,
> floppy drive access etc. I thought that such BIOS calls weren't
> accessible in protected mode (VM86 tasks aside), so how does it work?

aeBIOS is a native motherboard BIOS 32 bit protected mode interface.
There are no special calls or procedures required by the 32 bit protected
mode code to access the BIOS. Just stuff the registers and do the "int".
aeBIOS takes care of saving, restoring, converting and buffering.
The code is run in 32 bit protected mode, ring 0, 4Gb flat address space.
The A20 line is enabled. Interrupts are enabled and functional.
In this mode the segment registers need not and should not be explicitly
used. All "word" size BIOS parameters are 32 bit. All address pointer
BIOS parameters are 32 bit registers and the address can be anywhere in
available memory.

Nothing has been added to the MikeOs code to enable 32 bit protected mode
BIOS access, so in a sense the code has been "translated" to PM32.
The PM32 MikeOs code is a good example of how easy it is to program in
PM32 without the complexities associated with protected mode. Full BIOS
access is available without the restrictions of real mode.
