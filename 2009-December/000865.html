Hi Mike,

As regards the bin file uploader I'm coding for MikeOS, I've been thinking....

Instead of implementing all sorts of protocols etc which were made for transmission over modems
where data loss is a very real possibility, why not dispense with all the CRC and the likes, after all
what are the chances of data corruption while using a null modem cable of about a meter in length?
(From the development machine to the MikeOS machine next to it).

This will significantly reduce the size and complexity of the bin uploader.
It will also allow the sending of files from one MikeOS machine to another quite easily.(lot's of possibilities there).

So my question to you as follows:

Is there a distro independent utility in Linux that allows one to simply open a file in bin mode and just
send the bytes off to the serial port with no fancy protocols at a selected baud rate of say 9600 or 1200?

The reason I ask is that I will also provide source for an app to upload the compiled bin file for MikeOS when the development PC 
is either DOS or DOS under XP, but what I know about Linux is.......well next to nothing! 
(although LXF is helping, I just need the time to tinker with the penguin)!

Regards,
Paulo.
