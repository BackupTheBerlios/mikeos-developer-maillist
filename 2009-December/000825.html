> Perhaps someone could shed the light on the following:

> In MikeOS when text mode is called (screen 03h) it's has
> 80x25 characters with 16 colours.

That is for genuine CGA only, and is technically 640 x 200 giving 8 x 8
characters. This can be character dithered to give more pseudo-colours.

VGA emulates the mode as 720 x 400 and each character is 9 pixels wide
and 16 pixels high.
It is NOT a true representation of CGA as it will NOT dither to give more
pseudo-colours due to the odd pixel width per character.
My old IBM XT had genuine CGA and stuff I wrote for that broke immediately
on my later machines.

This is ancient stuff and is very well documented.

> When graphics mode is used (mode 13h) the screen resolution
> is 320x200 with 256 colours.

This can be 320 x 400 and was used extensively by Windows in the early
days of VGA(/MCGA) using VESA extensions and using the VGA memory
range, absolute addresses A0000 to BFFFF.

> Yet on my LCD screen, it reports the resolution as 720X400 at 70Hz
> It makes no difference whether the mode is set to text or graphics mode.
> Thinking maybe it was the LCD screen doing some up-scalling or something
> stupid, I connected a CRT monitor, pressed the OSD button and sure enough
> 720X400 at 70Hz.

It is correct as you are emulating CGA at VGA or greater level.

> Which brings me to my questions:

> 1) What resolution is mode 03h equivalent to ?

640 x 200 in text mode but only on genuine CGA cards.
(NOTE:- IBM had an extra mode for its genuine CGA card that could oly be
accessed via special SW that was graphics mode 160 x 200 in 16 colours and
WAS used by some games writers in the early 80s.)
720 x 400 on VGA and above.

This has been consistent for many, many years now.

> 2) Why 720x400, when mode 13h is 320X200 ?

It is probably emulating the mode internally but you are viewing it as 320 x 200.

its speed for pixellated DMA.
