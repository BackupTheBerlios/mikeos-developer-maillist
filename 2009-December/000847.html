I only use the AD86 combination for Windies and Windies DEBUG.EXE...

The memory mapping for, (assume absolute addresses in hex), CGA is:-

B8000 == character.
B8001 == colour attribute.
B8002 == character.
B8003 == colour attribute.
And so on...

To see what I mean open up a Command Prompt in Windows.
Execute DEBUG.EXE.

When in the debugger do this:-

db800:0000<RETURN/ENTER>

You will see the Microsoft Copyright etc, characters along with the atributes
07h which is normal white on black.

Use q<RETURN/ENTER> to quit DEBUG.EXE

Even addresses are the printable character, odd addresses are the
attributes therefore as for being unable to change colours use es:
in some simple assembly code for your own and write direct to the
first page CGA addresses, MikeOS does not crunch when going outside
of the 64KB constraint when using es:.

In a previous upload I mentioned I haven`t tried using ds: yet but, I do
suspect it might cause instability in the OS if I did. 
Obviously don`t change cs: AND ss: but so far es: is user friendly.
