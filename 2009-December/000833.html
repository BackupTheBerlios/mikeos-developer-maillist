Using PAROUT and PARIN LPT1 is too restrictive for the following reasons:

1) LPT addresses vary (see my help file)

2) The LPT port actually consists of 3 registers (data,status and control) with 3 different addresses
    The data port (base address is normally write only unless it has bi-directional capabilities
     and it must be switched by software to input mode by writing to bit 5 of the control register first.)
     (again see my help file)

3) A pc may have up to 3 LPTs so by hardcoding LPT1 is no good.


A much better way to go is:

OUT address,data       for 8 bit output ports such as the LPT
A = IN(address)          for 8 bit input ports such as the status register of the LPT

I also believe there should be the capability for 16 bit port access.
This would allow access to sound cards, PATA hard-drives , ATAPI cd-roms, etc.

OUTW address,data      data in this case is 16 bit
A = INW(address)        as above.

Just my 2 cents worth.
Paulo.

My idea is to wrap the accessing of a parallel port in PAROUT and PARIN
functions.

Example:

    PAROUT LPT1 , 12       <-- write 12 to the port
    PARIN LPT1, $A         <-- read port and store in A variable.
