> Wow! From a quick look, and perhaps I'm missing something here, the
> kernel still appears to use BIOS interrupts for screen handling,
> floppy drive access etc. I thought that such BIOS calls weren't
> accessible in protected mode (VM86 tasks aside), so how does it work?

The BIOS hasn`t really changed for decades, EXCEPT...

If you disassemble  FFFF:0000  the first instruction will ALWAYS be JMP F000:E05B,
this has never changed BUT the segment/offset HAS changed...

AFAIK if my memory serves me correctly, the segment still remains at 16 bit,
FFFF, but at the point of RESET the IP sits NOT at the 16 bit offset of 0000 but
32 bit offset of 00000000.
AFAIK on cold boot the x86 CPUs give an apparant 16bit/16bit segment offset
but in reality are 16/32 bit REAL mode representation of a pseudo_flat memory model...

I`m probably wrong as it was years ago since I looked into it...
