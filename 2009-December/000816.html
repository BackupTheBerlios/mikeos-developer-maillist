Bazza,

After reading your last post again, I saw you mentioned that you were using this for a ADC.
Many DAC's especially the TLP/TLC range from TI are quite finicky to timing when accessing them,
especially when using the SPI interface.

Many of the newer ADC's that use I2C are also very touchy about the bus timings and in general you
should limit I2C comms to 400KHz or less for reliable comms.

I've often used the parallel port to mimic the I2C protocol with no problems.
again make sure you use pull-ups on the lines which you are using as SDA and SCL.

Another thing is try and avoid using CMOS logic as it's terribly slow and can suffer from latch-up conditions.
Hence my recommendation in my help file to stick with the 74HC range of logic which offers the speed of TTL
but the low power consumption of CMOS.

Paulo.
